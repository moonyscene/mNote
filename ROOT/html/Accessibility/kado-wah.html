KADO-WAH 2.0 (웹 접근성 자동 평가 프로그램)

v2009.11.06 v2009.04.10 v2008.10.16

한국정보문화진흥원(KADO) 에서 제작한 Web Accessibility Helper(WAH)

웹 페이지의 접근성 준수여부를 평가하고 접근성의 오류들을 바로 잡아주는 수정 과정을 통해서 웹 개발자와 콘텐츠 제작자들에게 장애인들이 웹페이지 접근이 용이한 웹 사이트를 만들 수 있도록 도와줍니다.

  • 2004.12. KADO-WAH 최초개발
  • 2006.09. KADO-WAH 2.0 배포
    • ① 웹 콘텐츠 접근성 지침 세부항목별 준수율 여부 제공
    • ② 개발자 및 관리자가 원본소스 수정을 위한 FTP 업·다운로드 기능 추가제공
    • ③ 한 화면(레이아웃)에 검사된 소스, 체크항목 리스트 및 체크항목별 오류사항 수정방법을 동시에 제공
  • 2009.12. 차기버전 배포예정.

KADO-WAH 2.0 (patch1포함) 프로그램 다운로드

평가 마법사로 전체 평가할 때마다 결과가 조금씩 다르므로 적어도 2번 이상 평가 및 오류 수정을 반복합니다.
1회 평가시 "파라미터가 다른 동일한 파일의 URL 반복 제한" 항목을 "1번 반복.."으로 설정하여 검사하고 오류 수정합니다.
2회 이후에는 "10번 반복.."으로 설정 후 검사하여 오류 수정합니다.
오류 수정 후 최종본은 반드시 "URL 반복 제한 : 10번 반복..", "최대 페이지 수 제한 : 없음" 으로 평가해주세요.
 ※ 툴버그
* 웹 사이트 주소가 http://project.정보행동.co.kr:9100/ 형태로 포트번호가 있으면, 평가 마법사를 사용할 수 없다. (한페이지 검사만 가능하다.)
* ActiveX 가 포함되어 있을 때.. ActiveX 를 제거한 뒤 검사해보세요.

    평가 마법사 로그는..

    • 1페이지 평가 화면에서만 볼 수 있습니다. 로그 내용은 어느 페이지에서 보시든 자동검사 전체 페이지에 대한 로그입니다.
    • [평가 환경 설정] 에서 "접근성 오류 로그 파일 남김"을 설정하여야 텍스트 로그파일이 생성됩니다.
    • 로그파일이 저장 되는 위치는 프로그램이 설치된 폴더 하위의 Log 폴더입니다.
      일반적으로 C:/Program Files/KADO-WAH/Log/domain.go.kr_20081016_102002.txt 형태로 저장됩니다.
      KADO-WAH [로그 보기] 내용을 모두 복사하여 텍스트 파일(*.txt)로 저장하셔도 됩니다.
    • 로그파일 내용은 검사한 파일명, 참조된 외부파일 목록, [Rule 번호 오류 일람] 으로 구성되어 있습니다.
    • [Rule 번호 오류 일람] 은 오류, 경고, 수동검사 항목을 모두 포함한 내용이므로, [표1]의 Rule 만 수정하면 됩니다.

    항목 1.1 텍스트 아닌 콘텐츠(non-text contents)의 인식
    KADO-WAH Rule No.1 이미지에 대한 대체 텍스트(alt="X" 또는 longdesc="X")가 없습니다.
    <img> 요소에 alt 또는 longdesc 속성이 없을 때 보고됩니다.
    KADO-WAH Rule No.5 이미지맵에 대한 대체 텍스트가 없습니다.
    <area> 요소에 alt 속성이 없는 경우 발생합니다.
    KADO-WAH Rule No.6 이미지버튼에 대한 대체 텍스트(alt="X")가 없습니다.
    <input type="image">인 태그에 alt 속성이 없을 경우 보고됩니다.
    KADO-WAH Rule No.7 애플릿에 대한 대체 텍스트가 없습니다.
    <applet> 요소에 alt 속성 또는 대체 텍스트가 없을 경우 보고됩니다.
    KADO-WAH Rule No.9 개체(<object>)에 포함된 이미지에 대한 대체 텍스트가 없습니다.
    <object> 요소에 설명문이 없을 때 발생합니다.
    KADO-WAH Rule No.14 하나 이상의 이미지 맵 링크에 해당 텍스트가 없습니다.
    <area href=""> 요소에 alt 속성이 없는 경우 발생합니다.
    항목 2.2 프레임의 사용 제한
    KADO-WAH Rule No.13 프레임셋 내에 <noframes>를 사용하십시오.
    <frameset> ~ </frameset> 내부에 <noframes>가 없으면 보고됩니다.
    KADO-WAH Rule No.63 프레임에 제목이 없습니다.
    <frame>, <iframe>, <frameset>의 경우 title 속성이 없으면 보고됩니다.
    KADO-WAH Rule No.64 프레임 제목이 비어 있습니다.
    <frame>, <iframe>, <frameset>의 경우 title 속성이 있으나 그 내용이 비어 있으면 보고됩니다.
    KADO-WAH Rule No.65 <noframes>에 제목이 없습니다.
    <noframes> 태그에 title 속성이 없으면 보고됩니다.
    KADO-WAH Rule No.67 <noframes>에 FRAMESET 콘텐츠를 설명하는 대체 페이지에 대한 링크가 있어야 합니다.
    <noframes> ~ </noframes> 내에서 <a> 태그가 발견되지 않을 때 보고됩니다.
    항목 2.3 깜박거리는 객체 사용 제한
    KADO-WAH Rule No.48 <blink>를 사용하지 마십시오.
    <blink> ~ </blink> 요소가 있으면 보고됩니다.
    KADO-WAH Rule No.106 marquee 태그로 생성된 움직이는 텍스트를 사용하지 마십시오.
    <marquee>~</marquee> 태그가 있으면 보고됩니다.
