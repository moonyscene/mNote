WAMS - 케익소프트(CakeSoft)

v2010.02.19 진단 에러 이유 추가
v2009.09.03 유지보수 요청 받은 사천시 대표 웹사이트 검사결과 (접속기간~2009.09.30) 분석

WAMS 를 만든 케익소프트 에서 2006년 말부터 독자적으로 웹 접근성 진단하여 순위를 매긴 후 공공기관 등에 무차별적으로 결과를 배포하는 영업을 한 적이 있다. 그 때 케익소프트의 온라인 웹 접근성 진단 툴로 케익소프트 웹사이트를 검사하였더니, 70% 정도 밖에 나오지 않아서, 메일로 게시판으로 질문을 보냈었는데, 답변 조차 없었다. 한동안 잠잠하더니, 2009.09.03 에 또다시 무차별 영업 행태를 반복하고 있다. 이번엔 KADO-WAH 로 케익소프트 를 확인해보니 이제는 오류가 발견되지 않고 그럭저럭 신경 쓴 흔적이 보인다. 하지만, 검사결과 페이지 자체를 KADO-WAH 와 수동 점검해보았더니 대체텍스트, 데이터 테이블 등 오류가 다수 발견되었다. 정말 한심한 곳이다. 하지만, 앞으로도 이 업체와 계속 부딪혀야 하므로, WAMS가 얼마나 신뢰할 수 있는 진단 툴인지 점검해보았다.

오류 검출된 지침 내용 중 진단 에러는 (X) 로 표시하였다.

  • (O) 이미지에 "alt" 속성이 없습니다.
  • (X) 이미지에 "alt" 속성 값이 255자를 초과합니다.
    .. 현재 보조기술 수준으로 봤을 때 255자가 넘었다고 오류라고 단정하는 것은 억지다.
  • (O) 움직이는 이미지(애니메이션)에 "alt" 속성이 없습니다.
  • (X) <script>에 <noscript>가 없습니다.
    .. <noscript> 는 필요한 곳에만 있으면 된다.
  • (O) 프레임에 "title" 속성이 없습니다.
  • (X) "onmouseover"와 "onfocus"을 함께(pair) 제공하지 않습니다.
    .. 마우스와 키보드 운용이 가능하다면 onmouseover 와 onfocus 를 함께 제공하지 않아도 된다.
  • (X) "onmouseout"과 "onblur"를 함께(pair) 제공하지 않습니다.
    .. 마우스와 키보드 운용이 가능하다면 onmouseout 와 onblur 를 함께 제공하지 않아도 된다.
    같이 제공함으로서 운용에 또다른 오류가 발생할 수 있다.
  • (X) 자동 전환 페이지를 사용하고 있습니다.
    .. 자동 전환되더라도 非일반인 차별이라고 볼 수 없다.
  • (X) 데이터 테이블에 제목행과 제목열을 명시 하도록 합니다.
    .. 레이아웃 테이블도 있을 수 있다.
  • (X) 데이터 테이블에서 제목을 제공하는지 확인하십시오.
    .. 레이아웃 테이블도 있을 수 있다.
  • (X) 데이터 테이블의 크기는 상대 값을 사용하도록 합니다.
    .. 절대 값을 사용해야 할 경우도 있으면 그렇다고 웹 접근성이 떨어진다고 볼 수 없다.
  • (X) 머리글 요소의 사용이 적절하지 않습니다.
    .. 적절한 머리글 사용은 무엇인가? 머리글 요소와 웹 접근성은 관계 없다.

결론 - 진단 결과에서 alt 속성, title 속성 여부만 참고하면 되겠다.
