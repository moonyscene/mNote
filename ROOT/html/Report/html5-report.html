HTML5 마크업

  • 6.24일자 마크업 명세 - 확정상태라 봐도 무방
  • 하위호환성 유지, 간소화 정의(독타입, 스크립트/스타일 타입 생략..)
  • 현 데스크탑 브라우저에서 폼 등 상호작용요소의 지원률이 낮고, 최신버전 모두의 지원 후에도 하위 브라우저의 호환성을 유지해야 할텐데, 호환성을 위한 작업 가이드라인 필요
  1. 새로운 구조요소
    1. section, nav, article(본문), aside(본문과 무관한 내용), header, footer..
    2. ie에서 header의 선언만 유의하면 최신브라우저에서 사용가능
    3. default가 inline으로 간주
    4. section - article 간 종속성향 없음, 쌍방포함 가능
    5. 단지 스타일, 스크립트동작을 위한 래핑은 구조로서 의미없는 div사용
  2. 새로운 의미요소
    1. time(날짜와시간), b(중요성은 없지만 다르게 표현되어야 하는), i(어조가 다른.. 기술용어, 학명등)..
  3. 새로운 상호작용요소 : 현 브라우저 지원률 낮음
    1. input 추가 속성 : autofocus, placeholder(포커스 전 안내멘트, '소중한 의견을 입력해 주세요'등), required(필수입력사항, 서브밋 하면 자동 포커스 이동과 안내멘트 제공)..
    2. input type 추가 : 생략, 대략 훌륭함!
    3. progress(value/max 속성을 사용하는 진행상황), meter(min/max/value 속성을 사용하는 막대그래프), datalist(label처럼 id로 input과 짝, 자추찾는검색어 등의 참조보기&선택 리스트)..
    4. 멀티미디어 : embed부활, video(<source src="" type=""> 중복사용으로 불특정다수 브라우저에 맞게 다양한 지원포맷을 삽입할 수 있다), canvas(백터이미지)

  • 유효성?? 공식표준은 존중하되.. 라고 발표자는 말하지만..
  • 모바일을 위한 방향에서는 기획/시안디자인 부터 학습, 응용선행이 필요

  • 시험지원은 접두사를 선택자에 붙힘 : -o-(오페라), -moz-(파폭), -webkit-(크롬,사파리)

  • 발표자료참고 - 크롬 or 사파리로 보세요

HTML5 API

  • 미디어 컨트롤, 데스크탑 수준의 경험, 오프라인 지원, 리얼타임 푸시
  • 새로운 선택자 : getElementsByClassName, querySelectorAll
  1. Audio와 Video
    1. 속성 controls지정으로 브라우저 기본 컨트롤러 사용할 수 있음
    2. play(), pasue() 등 스크립트 제어와 미디어재생상황 이벤트 제공
  2. Canvas, SVG : 차트, 다이어그램 등 응용
  3. 데스크탑 수준의 경험
    1. Web Workers : 브라우저에 부담없는 연산 수행, 복수 동시 다발 사용
      var workerName = new Worker('**.js'); woekerName.onmessage 로 결과 받음
    2. Drag And Drop + File API : 외부창 파일 끌어오기 가능
    3. File API : 서버 송신 전 파일정보 분석/유효성 검사 등
    4. Geolocation : 사용자 위치 정보 파악/이용
    5. WebGL : 3D, 생략
  4. 오프라인 서포트
    1. Application Cache : 페이지내 핵심 js, css, image 캐시
      <html manifest=""> 로 캐시목록파일 지정
    2. Web Storage : 로컬 데이터 베이스, 쓰던 글 저장 등, 브라우저 종료후 유지 가능, 효율적 쿠키 대체 수단
    3. Web SQL Database
발표자 초청으로 현 진행중 프로젝트 감상 - Canvas, Web Workers등을 사용한 MRI단층 촬영 보기, 단층을 이동하며 빠르게 렌더링, Level을 제어하며 단층 표현 내용을 조정
