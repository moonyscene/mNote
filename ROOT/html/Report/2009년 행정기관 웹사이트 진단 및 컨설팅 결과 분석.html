진단 결과 재검토 요청 (진단 결과에 대한 이의)

웹 표준 진단 및 득점 산정 방법 문제

W3C Markup Validator 에서 진단 웹페이지의 오류 발생 비율이 5% 이상이면 무조건 0점 처리됩니다.
95%~0%의 표준 준수가 모두 같은 점수로 일괄 계산되는 것은 잘못된 진단이라고 봅니다.
CSS, Script 오류도 페이지당 오류가 1개라도 발생하면 0점 처리되는데,
코드 전체 라인 수 당 오류 라인 수의 비율로 산정하는 것이 더 정확한 진단이라고 생각됩니다.
※ Validator 오류는 일부를 제외하고 웹 접근성과 호환성에 그다지 영향을 주지 않습니다.
진단 점수의 고저 보다는 이 부분에 대한 정확하고 상세한 컨설팅이 필요하다고 봅니다.

진단 종합 (W3C 권고안 및 Validator Error 의미) - 갑에게 설명 필요!

W3C (X)HTML, CSS 명세는 W3C의 권고안 일뿐이며, 브라우저가 표준을 모두 지원하는 것은 아닙니다.
그리고, 표준 문법이라 하더라도 다음 권고안에서는 달라질 수 있습니다.
저희는 웹 접근성과 브라우저 호환성을 우선 고려하였고, 가능한 권고안에 맞추면서 작업을 하였습니다.

Error 라고 표시되는 것이 현재 웹페이지에 실제로 오류가 있다고는 볼 수 없으며,
권고안과 다르니 웹 접근성과 호환성에 유념하라는 의미라고 생각하시면 됩니다.
이번에 검출된 오류의 대부분은 웹사이트의 접근성, 호환성에 영향을 미치지는 않습니다.

W3C Markup Validator Errors 진단된 항목들

브라우저 호환성 고려하여 작업한 것들..

에러의 대부분은 아래 3가지 경우입니다.

  1. 특수문자(&)를 Name Code(&amp;) 형태로 사용하지 않은 경우.
    오류 발생의 소지가 있어서 Name Code 형태를 권고하고 있긴 한데,
    현재 통용되는 브라우저에서는 그렇지 않으며, 오히려 &amp; 사용시 간혹 문제가 발생하여서
    특수문자(&) 그대로 사용중입니다.
  2. 종료 태그가 없는 홀로 쓰이는 태그에 종료 표시를 해준 경우. <meta /> <link /> <base /> <col /> <area /> <frame />
    모든 시작 태그들은 종료 태그를 가져야 하는 것이 XHTML 표준 문법입니다.
    현재 도큐멘트 타입 HTML4.01 에도 문제가 없어서 상위 호환성을 위해서 XHTML 문법대로 코딩하고 있습니다.
  3. 비표준 속성 allowtransparency="true" 를 사용한 경우.
    현재 웹표준 권고안은 아니지만, 대부분의 브라우저에서 지원하는 속성입니다. 호환성을 위해 넣었습니다.
화면에 영향을 주지 않아 발견하지 못한 것들..
  1. img 요소의 필수 속성 alt 가 없는 경우.
  2. script 요소의 필수 속성 type 이 없는 경우.
  3. 요소에 없는 속성을 사용한 경우. onfocus, onblur, target 등..
  4. 같은 속성을 중복해서 사용한 경우. tabindex="0" 등..
  5. </a></li></a></li> 와 같이 잘 구성(well-formed)되어 있지 않은 경우.
  6. <ul> 하위에 <li> 없이 다른 요소를 사용한 경우.
  7. <a href="#"><a href="#"></a></a> 와 같이 중첩 앵커를 사용한 경우.
  8. 종료 태그 </div> </span> 등이 모자라거나 남는 경우.
  9. 태그의 속성 값들에 따옴표(" 또는 ')를 사용하지 않은 경우.
  10. ""(따옴표)를 중복으로 코딩한 경우.
  11. 주석 안에 불필요한 -(dash)가 있는 경우. <!-- <!-- -->
  12. document.write(); 함수 안의 마크업 코드가 문법 오류로 해석된 경우.
  13. <html></html> 외부에 자바스크립트 코드 등을 사용한 경우.
  14. 인라인 요소 안에 블럭 요소가 포함된 경우.
