2010.04.09
==>@@수신 : 전체(경영지원 제외)

제목 : 웹접근성 실태조사 결과 보고서 ★평가오류. 답변과 조치 안내

03.29 쪽지로 작업보류하였던 ★평가오류 에 대한 정보화진흥원 답변요약과 조치 안내입니다.
보완 조치 후 반드시 유지보수 게시판에 내역 남겨주시기 바랍니다. (처리 완료 여부 확인이 가능하도록..)
현재까지 결과 보고서 pdf 파일 보내준 곳은 11군데입니다. (유지보수 게시판에서 "접근성" 으로 검색)
정보화진흥원 답변요약 : 함안군청 사이트는 웹 접근성 준수가 매우 잘 되어 있는 사이트입니다.
그리고 준수율이 100%가 아니더라도 부적합이 아니라 일부 보완을 하면 좋을 것 같다는 의견정도로 보시면 됩니다.
▶조치 안내 : 갑은 정보화진흥원 답변처럼 생각안하므로 ㅡ..ㅡb 아래 내용대로 조치합니다.

★평가오류1: 이미지에 대한 대체 텍스트 제공이 미흡함
조직도 http://www.haman.go.kr/02administration/02_01.asp
==>이의 제기! area 요소의 alt 속성으로 대체 텍스트를 제공하고 있는데..
실수 인것 같다.. 죄송합니다.
▶area 요소 alt 속성 또는 longdesc 속성에 적절한 대체텍스트가 있는데도 불구하고 부적합 판정을 하고 있다면, 갑에게 정보화진흥원에서 실수한 거라고 알려주시면 됩니다.
★평가오류2: 이용에 시간제한이 있는 콘텐츠에 제어 기능 제공하나 작동되지 않음.
메인 팝업존 http://www.haman.go.kr/main/
==>이의 제기! 접근성 문제 없으며.. 심사자가 착각한것 같다..
잘 구현이 잘 되어 있습니다.. 정지 버튼의 기능이 클릭 이벤트가 아닌 포커스에 의해 상실되어 버리고 있기 때문에 정지 버튼의 기능에서 벗어났다고 할 수 있습니다. 그래서 약간의 감점을 되었습니다.
▶논란거리가 없도록 코딩팀에서 picpr.js 갱신(정지 버튼 제공시 포커스에 따라 동작 상태 변하지 않도록) 예정임. 이후 프로젝트는 2010.04 이후 버전을 사용바람.
 ※ 장애인에게만 차별을 두는 것이 아니므로, (심사자의 예상과 다르게 동작된다면) "사용성" 문제일뿐이지 "접근성" 감점 요인은 아니라고..다시 이의제기 중이지만 답변 기대 안함.

★평가오류3: 레이아웃 테이블에 부적절한 속성 또는 태그를 제공하고 있음.
관내사업추진현황 http://www.haman.go.kr/program/incsilboard/lstEnterprise.asp
==> 이의 제기! "레이아웃 테이블 summary 속성 제공" 은 접근성을 저하시킨다고 볼 수 없으며..
접근성에 대한 저하 문제는 많은 논란거리가 있긴 합니다만, ..중략.. 계도 차원에서 평가 항목에 포함을 시켰기 때문에 이번 조사 기준에 평가항목에서는 레이아웃 테이블에 적절한 summary 속성을 제공하였더라도 약간의 감점을 주고 있습니다.. 현재 품질마크에서는 관련 내용을 평가하지 않는 건 맞습니다. 허나 실태조사에서는 이 부분에 대해서 앞서 설명했듯이 일부 포함해서 평가를 했습니다.
▶지적된 곳만 summary="페이지 네비게이션" 을 삭제합니다.
그리고, 이후 프로젝트는 레이아웃 테이블에서 summary 속성을 제공하지 않습니다. 진보로 고통받느니 보수를 수용하는게 편할 수도 있습니다.
 ※ 다음 평가 혹은 조사에서는 어떻게 평가하는지.. 재문의 중이지만 답변 기대 안함.

★항목2. 멀티미디어 매체의 인식 준수율 90.0% 라고 되어 있는데..
어떤 부분이 부적합인지 나와 있지 않습니다. 알려주시면 보완하도록 하겠습니다.
게시판 페이지에 동영상이 링크되어 있습니다. 외부사이트에 있는 동영상이라고 할지라도 현 사이트에 콘텐츠를 제공하고 있기 때문에 평가대상이 되었습니다.
▶외부 콘텐츠는 외부 링크로 제공하고, 현 사이트로 가져오지 마세요.

2. 실팀별로 구축시 담당자(@기획자, @콘텐츠 디자이너, @프로그래머) 에게 전달합니다.
3. 구축시 담당자(없으면 대체 인력)는 본인 작업 영역을 파악한 후 유지보수팀과 배분하여 처리합니다.(유지보수 게시판에 처리내역 남겨주세요.)
4. 처리중 애로사항은 코딩팀에게 문의하시기 바랍니다.
