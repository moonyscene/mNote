  4. css : png24 이제는 익스6 상관없이 그냥 사용하면 되는건지 어떻게 사용해야 하는지 알려주세요
    ==>png24 투명 이미지 사용해도 되지만 IE6 (20130319현재 점유율 1% 내외) 도 확인하자! 투명 부분 IE6 호환 위해 필요한 CSS 는 비표준이므로.. 주석처리 해두자. 
    <img src="/img/c/ex.gif" alt="대체텍스트" class="png24" />
/* * html .png24{tmp:expression(setPng24(this));} */ 
function setPng24(obj){obj.width=obj.height="1";obj.className=obj.className.replace(/\bpng24\b/i,"");obj.style.filter=	"progid:DXImageTransform.Microsoft.AlphaImageLoader(src='"+obj.src+"',sizingMethod='image');";obj.src="";return"";}
    그리고, IE6 에서 확인하면, 투명 영역이 회색으로 보일거다. 콘텐츠 이용에 문제 없으면 그대로 두고..
    다른 콘텐츠를 가리거나 링크가 안되거나 하는 문제가 발생하면.. 다음 둘 중에서 선택하여 코딩한다.
    1) 대표포털 같은 웹 호환성 진단 대상 웹 사이트 : IE6 용 이미지 파일을 별도로 만들어서 적용한다.
    2) CSS 문법 통과 안해도 되는 사이트 : 비표준 CSS 적용해도 되지만, 링크 영역 오류 등이 발생할 수 있으므로 반드시 점검 필요!
    IE6,7 에서 PNG-24 이미지에 이미지맵을 걸면 안됨. (IE8은 가능)
    배경 이미지도 같은 기준을 적용한다. 단, 아래처럼 다른 비표준 CSS 를 사용하며 JS 는 필요없다.
    * html #m1 li a .mask{background:url(/img/all/b.png);filter:progid:DXImageTransform.Microsoft.AlphaImageLoader(src="/img/main/m2pmask.png",sizingMethod=crop,enabled=true);}
    PNG-24 배경 버그
    * 배치 고정 {backgroun-position:left top;}
    * 반복 불가 {background-repeat:no-repeat;}
    * [IE6,7] PNG-24 배경이 사용된 요소의 자손 요소에 링크가 있으면 마우스 접근 불가 (키보드는 가능, IE8은 가능)
    * [IE6] PNG-24 배경이 사용된 요소 위(z방향)에 투명한 a요소가 있으면, 불투명 배경 영역만 클릭되고, 투명 영역 클릭 불가
    기억할 것! 투명 png 남발하지 말자!
