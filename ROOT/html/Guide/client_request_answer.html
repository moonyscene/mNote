    유효성 검사

    W3C 권고안 일뿐이다. 브라우저가 표준을 모두 지원하는 것은 아니다. 현시점에서 가능한 문법에 맞춰 작업을 하였으며 브라우저 호환을 우선시 하여 작업을 하였다. 현재 문법이 다음 권고안에서는 달라질 수 있다.. 향후 행안부에서 웹 호환성 평가를 할 경우 좀 더 검토해보겠다. 2009년도 행정기관 웹사이트 진단에 웹 표준(호환성) 진단 포함

    자동 팝업

    메인 페이지 접속시 자동으로 팝업을 띄우는 경우..
    접근성 평가에서 감점된 사례가 있습니다.
    - 한국형 웹 접근성 지침 (KWCAG1.0) [http://www.iabf.or.kr/Lab/Kwcag/Kwcag.asp] [지침2-6 반응시간의 조절기능] 의
    체크리스트 : 사용자의 입력 없이 자동으로 발생하는 팝업창을 사용 하는 경우는 없는가? 에서 부적합 판정을 받게 됩니다.

    메인 페이지의 이미지 홍보(팝업존) 를 활용하시기 바랍니다.
    그래도, 팝업 형태가 필요하다면, 새 창이 아닌 메인 창에서 레이어 팝업으로 보여줘야 접근성에 문제가 없습니다.
    보여주는 방법도 있지만, 이로 인해 메인 콘텐츠를 가리거나 문서의 논리적인 구조가 고려되지 않음으로 인해 문제가 될 수가 있습니다.
    그리고, 브라우저의 팝업 차단 기능을 회피할 목적으로도 레이어를 이용하지 말고 메인 콘텐츠 안에 만드는 것이 최선의 방법입니다.

    #Ref. 웹접근성연구소 > 개발자 아카이브 > 웹 콘텐츠 제작기법 > 반응시간의 조절기능

    주소 표시줄 고정

    주소(URL)을 고정하기 위하여 프레임을 사용할 경우..
    접근성 평가에서 감점된 사례가 있습니다.
    - 한국형 웹 접근성 지침 (KWCAG1.0) [http://www.iabf.or.kr/Lab/Kwcag/Kwcag.asp] [지침2-2 프레임의 사용 제한], [지침3-2 논리적 구성] 에서 부적합 판정을 받게 됩니다.

    문제점들..
    불필요한 프레임으로 인하여 웹페이지 탐색에 방해가 된다.
    제목표시줄에 웹페이지 제목(title 태그 내용)을 표시하지 않는다.
    즐겨찾기에 추가를 하면 어떤 페이지에 있든 프레임 페이지 주소로 북마크된다.
    새로고침을 하면 어떤 페이지에 있든 프레임 페이지가 새로 고침된다.

    또한.. 주소 표시줄 고정, 상태표시줄 고정, 마우스 우클릭/드래그 금지를 보안 준수사항이라고 믿고 있는 사람이 많은데,
    이런 식의 사용자 브라우저 기능을 이용한 처리는 보안 수준을 향상 시킬 수 없고 웹페이지의 사용성만 저하 됩니다.

    텍스트 크기 고정

    [Q] IE 브라우저에서 텍스트 크기가 이상하다?

    IE 브라우저에서 [보기]-[텍스트 크기]-[보통]으로 설정하시면 됩니다.

    [Q] IE 브라우저에서 [텍스트 크기] 설정에 관계없이 고정시킬 수는 없는가?

    현재 body 태그에 적용되어 있는 기본 폰트 크기는 %(또는 em) 단위로 되어 있는데요..
    텍스트 크기로 민원이 계속 들어오면, 기본 폰트 크기를 px 단위로 수정하여 고정시킬 수는 있습니다.

    px 단위를 쓰면 웹접근성에 위배되는 것이 아니고,
    px 단위를 사용해도 표준을 잘 준수한 브라우저(파이어폭스 등) 에서는 텍스트 크기 조절 기능을 사용할 수 있습니다.
    버그가 많은 IE 브라우저에서도 텍스트 크기 조절이 가능하게끔 하기 위해 %(또는 em) 단위를 사용한 것입니다.

    그런데, 2007~2008 웹접근성 품질마크 심사에서 IE6 브라우저에서 텍스트 크기 확대축소가 불가능하면, 부적합 판정을 하고 있습니다.
    2009-01-22 게시글 "텍스트 크기 설정에 관한 문의드립니다."을 보면..
    사이트 운영 중 이 문제로 인해 잦은 문의나 불편 사항이 발생하는 점을 인정하여 웹 접근성 품질마크 심사에서는 글자 크기를 px로 고정하여도 인정하고 있습니다.

    - 한국형 웹 접근성 지침 (KWCAG1.0) [http://www.iabf.or.kr/Lab/Kwcag/Kwcag.asp]
    [1-3. 콘텐츠의 시각적 명료성] 지표 1-3-3. 텍스트는 쉽게 읽을 수 있도록 충분한 크기로 제공되어야 한다.
    체크리스트 : 사용자가 텍스트의 크기를 키우거나 줄일 수 있도록 텍스트 크기를 상대적인 단위로 적용하고 있으며, 크기가 변하더라도 내용을 인식할 수 있도록 바르게 표시되고 있는가?

    심사 판정에 문제가 있다고 생각하지만, 감점되지 않으려면 %(또는 em) 단위를 사용해야 합니다.

    • 글자크기로 계속적인 민원이 문제가 된다면, 홈페이지 이용안내 혹은 날개메뉴 등에..
      방송통신위원회 의 우측 "글자가 작거나 크게 보이세요?" 와 같이 이용법을 안내하는 것도 고려해보십시오.

    [Q] IE 브라우저에서 항상 [텍스트 크기]-[보통] 으로 설정되게는 할 수 없는가?

    사용자의 브라우저 설정을 변경하는 것은 불가능합니다.
    브라우저 설정과 웹사이트 내의 글자크기 확대는 전혀 다른 기능이라서 서로 설정상태를 공유할 수 없습니다.

    화면 확대

    화면 확대 기능 유무는 접근성 평가와 아무런 상관이 없습니다. (글자크기 확대도 마찬가지입니다.)

    #Ref. 확대, 축소, 인쇄 기능은 브라우저들이 기본적으로 지원하기 때문에 필수 기능이 아닙니다.

    알려진 문제점들이 많습니다.

    • 화면 확대는 IE 브라우저 전용 스타일시트인 zoom 속성을 이용하므로 IE 만 가능합니다.
    • 웹문서가 웹표준 마크업이면 확대할 영역에 스타일시트 { position: relative; } 를 지정하여야 합니다.
      (zoom 속성이 표준이 아니어서 그런지.. 비표준 마크업은 그냥 적용됩니다.)
    • 플래시 wmode 가 transparent 혹은 opaque 일 경우, 확대하면 플래시 무비 위치가 좌상단으로 이동됩니다. ☞ 진주시 - 비쥬얼 플래시
      테스트) 플래시 object 요소에 { position: relative; } 추가하여도 아무런 변화가 없다.. 해결 방안 찾지 못함.
      미봉책) 플래시를 화면 좌상단부터 시작되도록 크게 만든다. (무비가 커지고 다른 콘텐츠와 겹쳐져서 문제가 발생할 수 있습니다.)
    • 웹표준 마크업에서 확대 후 스크롤 막대를 움직이면 플래시 화면이 깨집니다. (wmode 와 무관) ☞ 진주시 - 비쥬얼 플래시
    • 비표준 마크업에서 확대 후 스크롤 막대를 움직여도 플래시 위치는 변하지 않습니다. (wmode 와 무관) ☞ 행안부 - 비쥬얼 플래시
    • 투명 PNG-24 이미지를 사용할 경우 투명영역이 검정색으로 보입니다. ☞ 진주시 - 좌측 서브 타이틀 이미지
    • 플래시 무비 ActionScript 에 Stage.scaleMode = "noScale"; 가 있으면 확대가 안됩니다.
      • ☞ 횡성군 … 테스트주소

    해결방법은 없습니다.

    웹사이트에 포함되는 화면 확대 기능 구현은 IE 브라우저만 가능한 비표준(권장하지 않는) 방법이므로,
    적용한 사이트도 드물고 문제점들에 대한 해결책 또한 없습니다.

    문제점 없는 화면 확대 기능을 구현하려면, 사이트 설계 단계부터 플래시와 투명 PNG-24 이미지가 없는 디자인으로 작업해야 합니다.

    화면 확대 기능을 별도로 제공할 필요가 없습니다.

    [표1]과 같이 주요 브라우저에는 화면 확대 기능이 있고 오류도 없으므로, 화면 확대 기능을 별도로 제공할 필요가 없습니다.
    (익스플로러7이상, 파이어폭스3.x, 오페라9.x 브라우저 등에서 화면 확대 기능을 제공중입니다.)
    그래도, IE6 사용자를 위해 필요하다면, 많은 문제점들을 안고 화면 확대 기능을 제공할 수 밖에 없습니다.

    대응 방안

    IE6 비표준 브라우저라서 표준 코드를 제대로 인식하지 못하여 발생합니다.
    IE7, IE8, FF3.x, Sf3.x, Op9.x 에서 확인해보면 이상없이 잘 인쇄됩니다.
    IE6 에서도 인쇄를 요구할 경우, CSS를 다른 방법으로 전환해 보면 해결책을 찾을 수도 있습니다.

    인쇄 문제! 원인에 따른 증상과 해결책

    * KRX 국문 - 지수 - 지수안내 - KOSPI계열 - KOSPI
    증상) [IE8] { position: relative; } 요소와 자손요소{ float: none; } 인쇄안된다.
    해결) { position: relative; } 속성 제거 후 인쇄된다.
    과제) 자손요소{ float: left; } 경우는?

    * 부산시 -- 프로필
    증상) { float: left; } 요소들 부모에 { width: 값; overflow: auto; } 없으면 인쇄안된다.
    해결) 부모에 { width: 값; overflow: auto; } 둘다 지정한 후 인쇄된다.
    하지만 콘텐츠 그룹이 길어서 1페이지에 많은 공백을 남기고 2페이지로 넘어가서 인쇄되는 문제가 있다.

    * 부산시 -- 채용시험
    증상) { width: 값; overflow: auto; } 가 있고 자식요소는 플로팅이 아닐때 { float: none; } 인쇄안된다.
    해결) width, overflow 속성 중 1개 이상 제거하면 정상적으로 인쇄된다.
    ※ 위지윅 비표준 코드, float, js로 display 표시/비표시 등에서 인쇄 문제가 발생한 것으로 생각했으나 아니었음..

    * 함안군 어린이 군청 - 군청에서하는일
    증상) { float: right; } 요소 다음 콘텐츠가 인쇄안된다.
    해결) 플로팅 다음 요소에 { style="clear: both;" } 추가하면 된다.

    * 사천시 본청, 어린이 … 작업주소
    증상) 사이드바가 본문 인쇄시 나오는 문제가 발생하였다. (원인: 인쇄용 CSS 지정순위)
    해결) <head></head> 의 마지막에 인쇄용 CSS (print.css) 를 참조하고, 인쇄용 CSS 속성은 !important 를 붙인다.

    /* 본문인쇄용 */ @media print { #head, #visual, #sidebar, #body_foot, #foot, #wing, #fontsize { display: none !important; } body, #container, #wrap, #body, #body_head, #body_content { margin: 0 !important; padding: 0 !important; border: 0 !important; background: #fff !important; } }

    * 이순신 관광지도 … 작업주소
    증상) 인쇄가 다르다~. 그냥 페이지, iframe 페이지. (FF에서는 똑같이 인쇄됨.)
    해결) 2008.10.22 사이트 접속 불가라서 다음에~.

    문제점들

    Internet Explorer 이외의 다른 대중적인 브라우저(Firefox, Opera, Safari, Chrome)에서는 플래시 메뉴에 키보드 접근이 되지 않는 호환성 문제가 발생합니다. 이것은 플래시 메뉴 제작자의 잘못이 아니라 플래시 플레이어를 제작하고 배포한 Adobe사가 다양한 브라우저에 대한 키보드 접근행태를 고려하지 못한 상황으로써 이 문제가 해결되기 전까지는 글로벌 네비게이션과 같은 핵심 콘텐츠에 플래시를 사용하는 것을 권장하지 않습니다.
    #Ref. 웹 접근성 자문

    IE에서도 플래시 콘텐츠의 wmode가 transparent(투명)일 경우 Tab키로 플래시에 접근을 할 수 없는 등 자체적인 접근성은 아직 미흡한 부분이 있습니다.
    (2008.11.13 현재 IE7이상만 가능)

    상단 주메뉴를 플래시로 구현한 사이트 를 오페라 브라우저의 [보기]-[작은 화면]으로 보면 플래시 메뉴 글자가 작아서 읽을 수 없습니다.
    PDA, 휴대폰 소형기기에서는 제대로 볼 수 었습니다. (Flash를 지원안하거나 지원하더라도 메뉴 글자가 작아져 선택이 어려움)
