01-03. 웹 표준

  • 전체 웹 페이지가 Markup(HTML, XHTML)과 CSS 유효성을 통과하도록 노력하자!!
    (브라우저의 표준 지원 정도가 달라 유효성 통과가 불가할 경우 최소한의 비표준 코드는 허용)
  • div 와 CSS 를 이용해서 레이아웃을 잡고 시맨틱(의미에 맞는) 마크업을 하는것.
  • 구조(HTML, XHTML), 표현(CSS), 동작(JavaScript, DOM)을 분리해서 코딩하는것.

※ 참고 : 실전 웹 표준 가이드 (2005.12), 전자정부 웹 표준 준수지침 (행안부, 2008.04), 웹 표준 (직접 정리)

02. 웹사이트 기본 구조

02-01. HTML 문서 기본 구조

<!DOCTYPE html> <html lang="ko"> <head> <meta charset="utf-8" /> <script>/*<![CDATA[*/ var $meta = { title: '사이트명 - 문서제목</title> <meta name="author" content="저작자"> <meta name="keywords" content="문서제목, 키워드, 키워드, 저작자"> <meta name="description" content="문서설명을 서술한다. 또는 제목"> <link rel="stylesheet" type="text/css" href="/share/css/print.css"> </head> <body> <h1>문서제목</h1> </body> </html>

행안부(구 행자부)에서 실시한 "2007년 행정기관 웹사이트 진단 및 컨설팅" 의 기술평가에서
meta 태그 중 keywords, description, author 의존재 여부와 external CSS (link 태그) 사용 여부를 체크하므로
모든 웹문서는 위 기본 구조처럼 meta 3가지, link, h1을 반드시 포함하도록 한다.
※ "/share/css/print.css" 는 본문 인쇄용 스타일시트(화면 디자인 영향 없음)이므로 css 가 필요하지 않는 longdesc 등에도 사용하면 된다..

적용 샘플

<!DOCTYPE html> <html lang="ko"> <head> <meta charset="utf-8" /> <script>/*<![CDATA[*/ var $meta = { title: '(주)정보행동 - 인사말</title> <meta name="author" content="(주)정보행동"> <meta name="keywords" content="인사말, (주)정보행동, 정보행동, 웹에이전시, 기업 부설 연구소"> <meta name="description" content="웹에이전시 (주)정보행동 인사말입니다."> <script src="/share/js/all.js?2008040816124"></script> <script src="/share/js/main.js?2008040816124"></script> <link rel="stylesheet" type="text/css" href="/share/css/print.css"> </head> <body class="d1n0 d2n0 main"> <h1>인사말</h1> <p>안녕하십니까?</p> </body> </html>

02-02. 네이밍 /폴더/파일명, 레이아웃 - id/class 선택자 선언 규약

02-03. myshin Library Guide

  • /share/js/all.js : 참조파일 캐시 안불러오기, css, js 파일 포함 방법, js 그룹별 파일 분리.
  • 예외 환경( Disable CSS, JS, Plugin, Images )의 유저에이전트(브라우저) 고려.
  • 코드 라이브러리(템플릿), HTML 수정 없이 sub.css, board.css 만 수정하여 디자인 적용
    • /_lib/sub/ : 본문 내용 → (예)
    • /_lib/board/ : 멀티게시판/회원로그인 → (예)

02-04. (X)HTML, CSS, JavaScript 코딩 방법들..

  • 시맨틱 마크업 - 의미에 맞는 적절한 HTML태그를 사용 : title, h1~h6, ul, ol, dl, p 등
  • 링크(a, area) : href="" 에는 링크(uri)만 적고 onclick="" onkeypress="" 로 동작을 수행
    tabindex="0" 는 가급적 사용말고 마크업을 논리적 순서로 구성.
  • 플래시, 동영상 은 접근성을 고려하여 object 태그로 직접 코딩한다.
  • 대체텍스트 : alt 속성은 대체텍스트 그대로, 부연 설명이 필요하면 title, 구조적 태그가 필요하면 longdesc 속성을 이용하자.
  • CSS : float, position 화면표시모델에 대해 완벽하게 이해하자. ※참고자료[Book]
  • 레이아웃 : CSS와 div 위주로 구현하고, 배치용 table 은 사용하지 않도록 한다.
  • 데이터 테이블 : summary 속성에 표 요약 내용을 적고, 표 제목은 caption 또는 h1~h6 태그로 코딩한다.
  • noscript 태그는 필요한 곳에만 있으면 된다.
  • JavaScript : 예전 비표준(document.all.객체명 ..) → 표준 DOM 스크립트 사용 (document.getElementById("객체명"))
    자바스크립트 개발 혹은 적용시 반드시 다음 조건을 만족해야 한다.
    • IE외 브라우저에서도 무리없이 동작되어야 한다.
    • 키보드 만으로도 동일한 기능을 수행할 수 있어야 한다.
    • 자바스크립트 제거시에도 링크, 버튼, 서식이 올바로 동작해야 한다.
