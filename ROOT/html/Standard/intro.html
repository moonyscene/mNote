블럭 요소 vs 인라인 요소
* <body> 에는 block 요소만 가능 * block 요소 안에는 block 요소, inline 요소 둘다 가능 * inline 요소 안에는 inline 요소만 가능 * <p> 안에는 block 요소가 불가 * <blockquote> 는 block 요소만 가능 * <ul>,<ol> 안에는 오직 <li> 만 가능 * <li> 안에는 뭐든지 가능
  • 인라인 요소
    a abbr acronym b basefont bdo big br cite code dfn em font i img input kbd label q s samp select small span strike strong sub sup textarea tt u var
  • 블록 요소
    address blockquote center dir div dl fieldset form h1 h2 h3 h4 h5 h6 hr isindex menu noframes noscript ol p pre table ul
  • 다른 블록 요소들을 포함할 수 있기 때문에 블록 요소로 간주되는 요소
    dd dt frameset li tbody td tfoot th thead tr
  • 블록 요소 또는 인라인 요소 모두로 쓰일 수 있다. 만일 인라인 요소로 사용되었을 때는 블록 요소를 포함할 수 없다.
    applet button del iframe ins map object script
태그 금지사항
  • a 태그 : 다른 a 태그들을 포함 할 수 없다.
  • pre 태그 : img, object, big, small, sub 또는 sup 태그들을 포함 할 수 없다.
  • button 태그 : input, select, textarea, label, button, form, fieldset, iframe 또는 isindex 태그들을 포함 할 수 없다.
  • label 태그 : 다른 label 태그들을 포함 할 수 없다.
  • form 태그: 다른 form 태그들을 포함 할 수 없다.
문서들이 잘 구성(well-formed)되어야 한다.
<p>여기에 강조 된 <em>문단이 있다.</p></em> (X) <p>여기에 강조 된 <em>문단</em>이 있다.</p> (O) [목록태그의 올바른 마크업] (X틀린예) <ul> <li>1차 목록 제목</li> <ul> <li>2차 목록 제목</li> </ul> </ul> (O맞는예) <ul> <li>1차 목록 제목 <ul> <li>2차 목록 제목</li> </ul> </li> </ul>
embeded CSS 는 오직 head 태그 안에만 위치할 수 있다.
<head> <style type="text/css"> </style> </head>
script 태그의 속성에 을 항상 명시해주고, language="Javascript" 속성은 삭제한다.
<script language="Javascript"></script> (X) <script language="Javascript"></script> (X) <script></script> (O)

XHTML 1.0 Strict

모든 태그이름, 속성이름은 소문자로
<img src="/img/b.gif" width="180" height="40" alt="" />
모든 시작 태그들은 종료 태그를 가져야 함. 빈(Empty) 태그 끝에는 공백/>
<area /> <br /> <col> <frame /> <hr /> <img /> <input /> <link /> <meta /> <param /> ..
태그의 속성 값들은 꼭 따옴표(" 또는 ')를 사용해야만 한다.
<img src=/img/b.gif width=180 height=40 alt= /> (X) <img src="/img/b.gif" width="180" height="40" alt="" /> (O)
속성은 속성이름="값" 처럼 짝으로 항상 완전하게 기술되어야 한다.
<input type="radio" value="라디오단추" checked="checked" /> <option value="선택상자" selected="selected"></option> 다음 애트리뷰트들이 관련된다.
checked, compact, declare, defer, disabled, ismap, multiple, noresize, noshade, nowrap, readonly, selected
ampersand(&), less-than sign(<) 은 Name Code(Number Code) 형태로 사용해야 한다.
& → &amp; < → &lt; 제어구문의 &(and)는 당연히 아니므로 찾아바꾸기 할 때 주의한다.
<!-- -->주석문 안에는 -(hyphen)이 여러개(-----) 올 수 없다.
<!-- 주석문 ---------- --> (X)

CSS Coding Hint

색상코드는 #을 반드시 붙여주세요.
border: 1px solid #ddd; background: #fff; color: #000;
숫자값에는 반드시 단위를 넣어주세요.
px : Pixels. (화면 해상도에 대한 상대 크기) % : Percentage. (자기가 속해 있는 상위 Tag의 크기 또는 상속받은 값에 대한 100분율 상대 단위) em : The height of the element's font. (지정되거나 상속 받은 font 크기의 상대 크기) - 현재 글꼴의 대문자 M 너비에 대한 상대 크기 px : The height of the letter "x". (소문자 "x"의 높이에 대한 상대 크기) 예) 100% = 1em
