유지보수성과 간결성이 상호배타적일 수 있다.

currentColor border-color, text-shadow, outline-color 등 많은 CSS 색상 속성의 기본 값.

inherit 계산된 부모 요소의 값 상속. 어떤 CSS 속성에도 사용할 수 있다.


input, select, button { font: inherit; }



숫자가 아닌 눈을 믿어라!
수치상 정중앙보다 약간 위에 있을 때 사람의 눈에는 중앙에 있는 것처럼 보인다.
상하 패딩을 좌우 패딩보다 적게 두어야 균일하게 받아들일 수 있다.

미디어 쿼리의 한계는 특정 장치에서 동작하지 않아서가 아니라 디자인 그 자체다.

단축 속성을 현명하게 사용하라.
background: 와 background-color: 는 같은 것이 아니다.

전처리기를 사용해야만 하나?
LESS, Sass, Stylus 와 같은 CSS 전처리기는 변수, 믹스인, 함수, 중첩, 생상 변환 등 CSS 제작에 편의를 제공한다.
적절히 사용하면.. 대규모 프로젝트에서 코드를 좀 더 유연하게 관리할 수 있다.

전처리기의 문제점

    

  • CSS 파일 크기와 복잡성에서 길을 잃을 수 있다.
    • 

  • 디버깅이 더 힘들어진다. ))) 이러한 문제를 줄이고자 나온 SourceMaps 에서는 생성된 CSS 코드와 본래의 CSS 코드를 매칭시켜 알게 해줌으로써 디버깅을 지원한다.
    • 

  • 개발 프로세스에 지연시간이 추가되었다.
    • 

  • 모든 것을 추상화 시킴으로써 다른 누군가가 코드베이스를 쉽게 사용하기 어려워졌다.
    • 

  • 협업자를 고르는데 제한이 생기고 사용법을 가르치기 위한 시간이 필요한데, 이 모두 최선의 선택은 아니다.
    • 

  • 구멍 난 추상화 법칙(Law of Leaky Abstraction)을 잊지 말아야 한다. 작은 것까지 모두 추상화한다면 빈틈이 생기기 마련이다. 전처리기는 인간이 개발한 것으로 자체의 버그가 있을 수 있다. 전처리기의 버그는 CSS의 문제의 뒤에 숨은 진짜 범인 일 수 있다.
    • 



여기에 나열한 이슈뿐 아니라 전처리기는 개발자들이 전처리기에 의존한 나머지 작은 규모의 프롲게트와 같이 불필요한 사용을 유도한다.
다수의 전처리기를 연상시키는 기능들이 이미 순수한 CSS에도 많이 만들어져 있다.

    

  • CSS Custom Properties for Cascading Variables
    • 

  • calc()
    • 

  • color()
    • 

  • CSS 작업 그룹은 중첩(nesting)에 관한 심도 있는 논의를 진행하고 있으며 이미 예전부터 규격 초안(ED)에 포함하고 있다.
    • 



순수한 CSS 로 모든 프로젝트로 시작하였으나 코드를 DRY 하게 유지하는 것이 힘들다면 그때 전처리기를 사용하기를 권한다.
의식 있는 결정을 하기를 바란다.
