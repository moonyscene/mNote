    데이터베이스를 사용하지 않는 블로그를 만들기 위해.. 정적 사이트 제너레이터 (static-site generator : 레이아웃 코드를 재사용해 빠르게 정적 사이트를 만들 수 있게 해주는 툴) 사용. 그 중에서 Jekyll 은 블로그를 위해 만들어진 프레임워크다. Jekyll 만의 약간은 고집스러운 스타일이 반영된 파일 수조를 기반으로 페이지나 글을 만든다. 사실 워드프레스나 대형 포털 블로그를 사용하는 블로거들을 위한 것은 아니다. 하지만 단순하고 효과적인 레이아웃 시스템을 가지고 있어, 테이터베이스에 대한 부담 없이 빠르고 간단한 블로그를 만들고 싶은 전문 블로거에게는 매우 적합하다.

    Jekyll 설치하기 -- "부록 A1 루비 설치하기" 먼저하자! 
    gem install jekyll
    Jekyll 는 동적으로 페이지를 만들기 위해 Liquid template language 를 사용한다. [m]https://github.com/Shopify/liquid/wiki/Liquid-for-Designers

    나중에 Jekyll 이 인식해 페이지 앞부분에 들어갈 내용을 추가할 수 있게 해 준다.
    YAML 이란 사람이 읽을 수 있는 형태로 데이터를 저장하는 포맷으로 JSON 과 같이 프로그래밍 언어 전반에 걸쳐 사용할 수 있다.

    2) CoffeeScript 인터프리터를 설치 보통 Node.JS, NPM(Node Package Manager) 과 함께 설치한다.
    루비와 함께 쓸 수도 있다. 루비를 설치했다면.. 다음 명령어를 커맨드라인에서 실행하여, CoffeeScript 인터프리터 와 Guard 를 설치한다.
    $ gem install coffee-script guard guard-coffeescript
    Guard 는 파일 변경을 관찰하다가, 바뀌면 미리 지정된 작업을 처리하도록 설정할 수 있는 커맨드라인 툴이다.
    프로젝트의 최상위 폴더에 Guardfile 이라는 이름의 파일을 아래 커맨드를 통해 만들 수 있다.
    $ guard init coffeescript
    새로 만들어진 Guardfile 을 열어 input, output 폴더 값들이 우리가 만든 폴더를 가리키도록 수정한다.
    # A sample Guardfile
# More info at https://github.com/guard/guard#readme
guard 'coffeescript', :input => 'coffeescripts', :output => "javascripts"

이 부록에서는 윈도, OS X, 우분투에서 각각 어떻게 루비를 설치하는지 살펴본다.

A1.1 윈도에 설치하기
루비 인스톨러 1.9.2버전 다운로드 후 C:\ruby 폴더에 설치.. 루비 인스톨러가 루비를 PATH 에 추가할지 물으면 추가한다. 이 방법으로 루비와 여러분이 설치한 다른 라이브러리들이 어떤 폴더의 위치에 있든 실행할 수 있다.
개발 키트 - 몇 가지 컴포넌트가 C언어로 작성되어 있기 때문에.. 필요한 컴파일러들이 들어 있다. C:\ruby\devkit 폴더에 압축을 푼다.
설정을 위해 명령 프롬프트 창을 열어 다음과 같이 입력하자. 
C:\ruby> cd \devkit
C:\ruby\devkit> ruby dk.rb init
C:\ruby\devkit> ruby dk.rb install
설정 테스트 위해, 명령창을 열어 cucumber 젬을 설치해보자. 
C:\ruby\devkit> gem install cucumber
이것으로 다 됐다. 이제 Cucumber 나 Sass 같은 라이브러리를 개발 프로젝트에 사용할 수 있다.
A1.2 루비 버전 관리자로 맥 OS X 와 리눅스(우분투)에 설치하기
루비 버전 관리자 (Ruby Version Manager, RVM) 를 이용해 루비 설치판 을 설치하고 관리
OS X 에 루비 버전 관리자 설치하기
Xcode, Git 설치 필요하다. 다음은 루비 버전 관리자 설치하기 $ bash < <(curl -s https://rvm.beginrescueend.com/install/rvm) 다음 명령어로 새로 터미널 세션을 켤 때마다 루비 버전 관리자와 파일들을 사용 가능하게 하자. 
$ echo '[[ -s "$HOME/.rvm/scripts/rvm" ]] && source "$HOME/.rvm/scripts/rvm"' >> ~/.bashrc
마지막으로 터미널 세션을 끄고 다시 시작해 루비 버전 관리자가 실행되도록 한다.
우분투에 루비 버전 관리자 설치하기
컴파일러, Git 외에 필요한 라이브러리들의 설치가 필요하다. 
$ sudo apt-get install build-essential bison openssl libreadline6 libreadline6-dev curl git-core zlib1g zlib1g-dev libssl-dev libyaml-dev libsqlite3-0 libsqlite3-dev sqlite3 libxml2-dev libxslt-dev autoconf
다음은 루비 버전 관리자 설치하기 
$ bash < <(curl -s https://rvm.beginrescueend.com/install/rvm)
다음 명령어로 새로 터미널 세션을 켤 때마다 루비 버전 관리자와 파일들을 사용 가능하게 하자. 
$ echo '[[ -s "$HOME/.rvm/scripts/rvm" ]] && source "$HOME/.rvm/scripts/rvm"' >> ~/.bashrc
마지막으로 터미널 세션을 끄고 다시 시작해 루비 버전 관리자가 실행되도록 한다.

[m] 설치 안된다 나옴. 안내된 명령어로 다시 설치함. 
sudo apt-get install ruby-rvm
루비 버전 관리자로 루비 설치하기
$ rvm install 1.9.2

[m]ERROR

$ rvm use 1.9.2 $ rvm --default use 1.9.2 Cucumber 라이브러리 설치하는 것으로 루비 설치 완료 테스트 할 수 있다. 
$ gem install cucumber
That’s it! Ruby and its prerequisites are now installed, and you can now use tools like Cucumber, Sass, Guard, and Jekyll in your web development projects.
