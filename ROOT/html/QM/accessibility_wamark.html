** 작업요강

목표 : [표2] 정통부 2007년도 웹 접근성 실태조사 결과의 13개 항목 모두 준수율 90% 초과하도록 보강.

※ 접근성 준수율 낮은 항목들 집중 보완 : 02 (1-2.영상매체의 인식), 12 (3-3.온라인 서식 구성), 13 (4-1.신기술의 사용)

ㅁ 프로그래머 :
  • 12 (3-3.온라인 서식 구성)
    3-3-1. 각 서식 제어 요소를 설명하는 레이블을 사용하여야 한다.
    3-3-2. 키보드만으로도 온라인 서식 입력 및 서식 간 이동이 가능하여야 한다.
  • 13 (4-1.신기술의 사용)
    4-1-2. 부가 애플리케이션이 작동하지 않을 경우에도 동일한 내용을 전달할 수 있는 대체 콘텐츠를 제공하여야 한다.
    → javascript 없이도 서식 이용이 가능하도록 수정한다.
    • ActiveX 가 있을 경우 IE이외의 브라우저는 ActiveX 없이도 해당 과제를 수행할 수 있도록 수정한다.(불가능할 경우만 보류).
    • 서식 이용 단계별 페이지 (ActiveX 설치 안내 등..) 에서도 대체텍스트 등 접근성 보강한다.
    • 통합검색(select 와 javascript 로 구현된)의 수정 작업은 일단 보류한다.(2008년 프로젝트부터는 접근성 고려하여 코딩한다.)
    • 부산 : 날씨 플래시에 대체텍스트(http://www.630.co.kr/popup/iconlist_1_1.htm 참고) 삽입(작업 방법은 이미 전달했음).
  • 10 (3-1.데이터 테이블의 구성)
    멀티게시판에서 코드의 table 요소에서 summary 속성을 보완한다.
    (예) summary="목록" 처럼 부실한 요약 설명을 아래 예들을 참고하여 수정한다.
    summary="공지사항 목록"
    summary="공지사항 게시물 목록을 볼 수 있고 제목 링크를 통해서 게시물 상세 내용으로 이동합니다."
    summary="각 게시물의 번호, 제목, 파일, 작성자, 작성일, 조회를 볼 수 있고 제목 링크를 통해서 게시물 상세 내용으로 이동합니다."
ㅁ 디자인코더 :
  • 본청 메인 페이지
    모든 접근성 항목(13개)에 대해 보완한다. (유지보수로 변경된 콘텐츠.. 이미지홍보(팝업존), 팝업 창 등 점검)
  • 13 (4-1.신기술의 사용)
    4-1-2. 부가 애플리케이션이 작동하지 않을 경우에도 동일한 내용을 전달할 수 있는 대체 콘텐츠를 제공하여야 한다.
    → 플래시, 동영상 등 object 요소를 페이지에 직접 코딩(javascript에 의존하지 않도록)하고 대체콘텐츠를 보완한다.
  • 02 (1-2.영상매체의 인식)
    동영상 원고 자료가 없어서 자막(원고)을 제공하지 못하더라도, 비동기 자막(원고)을 제공할 공간은 마련해둔다.
ㅁ UI코더(문영신, 황지영) : (참고)칠곡
  • 본청 메인 페이지에서 13 (4-1.신기술의 사용) 항목에서 아래 지표만 보완
    4-1-2. 부가 애플리케이션이 작동하지 않을 경우에도 동일한 내용을 전달할 수 있는 대체 콘텐츠를 제공하여야 한다.
    → javascript제거시 고려 :본청 메인, 서브의 html_head.* 에서 all.css, main.css 를 javascript로 호출하도록 수정한다. (아울러 print.css 도 추가).
  • iezn_embed_patch.0.43.js (ActiveX 이올라스 특허 문제 해결 스크립트) 추가한다.
