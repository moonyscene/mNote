품질마크 획득 지원 방법

1. 사전설문지 작성 지원

갑(클라이언트)이 작성가능한 부분을 기본으로 작성하고,
해당 과업 내용을 가장 잘 아는 기획자와 콘텐츠 디자이너 중 1명이 나머지 부분을 작성합니다.
코더가 사이트 구축 참여자들과 협의 수정하여 최종본을 만듭니다.

★ 품질마크 사전설문지 작성 요령 바로가기


#품질마크 신청 사이트. 사전설문지 작성자와 검토자 - 구축 당시 주요 참여자

  • #강서구. 작성(갑/손정은)검토(신) - 문영신,박미진,이녕길,손정은
  • #김해시. 작성(갑)검토(신) - 문영신,주인선,김수진,강광일
  • #남해군. 작성(갑/심윤정)검토(준) - 방문준,이미숙,김수진,심윤정
  • #삼척시. 작성(갑/김호정)검토(석) - 이영석,김호정,권용현,조창완
  • #창녕군. 작성(갑/김호정)검토(신) - 김호정,남태형,황지영,강광일
  • #청송군. 신청안함
  • #통영시. 작성(갑/이영석)검토(신) - 문영신,이영석,남태형,김대성
  • #함양군. 품질마크만료되어 이번에 신청할지 모르겠음. - 문영신,정유진,남태형,옥규화
  • #횡성군. 작성(갑/정창신)검토(신) - 문영신,김기리,남태형,정창신
  • #부산교통공사. 신청예정. 심사결과 지적사항 나오면 작업 지원 예정.

※ 코딩팀 : 문영(신),방문(준),이영(석)
※ 구축 당시 참여자가 없을 경우 각 실별로 작업자 배정해주시기 바랍니다.

2. 사전심사 대비 보완

한국정보화진흥원에 품질마크를 신청하면 언제 사전심사를 할지 모르니,
먼저 사전 심사 항목을 점검하여 보완 완료한 후~
사전설문지 최종본을 기획자를 통하여 갑에게 보내주시기 바랍니다.
** 사전 심사 : 자동평가(KADO-WAH)와 수동심사 병행
* 심사 항목 (적합기준 %이상) - 작업자 : 작업 내역
  • 대체텍스트(90%) - 콘텐츠 디자이너 : KADO-Wah 자동 평가
    ※ 주의 : 본청 하위 폴더에 존재하는 사이트로 인하여 90%이상 안나오면 하위 도메인으로 변경 등의 대책을 클라이언트에 제시하자!!
  • 테이블의미태그(70%) - 콘텐츠 디자이너 : 데이터 테이블은 summary, caption, h1~h6 중 하나라도 제공, thead, tbody, th 태그 사용.
  • 온라인서식레이블(90%) - 프로그래머 : label의 for 속성과 id값 연결
  • 페이지타이틀(70%) - UI코더 : 확인하시오.
  • 스킵네비게이션(50%) - UI코더 : 확인하시오.

사전 심사에서 부적합 받으면 전문가 심사도 진행하지 않고 평가 결과 보고서도 받을 수 없으므로 반드시 작업하도록 하세요!!

3. 전문가 심사 대비 보완

전문가 심사 (09.10.10~09.11.28) 시작 전에 사이트 구축 당시 참여자들은
웹 접근성 가이드라인 18개 지표 모두 준수 기준에 적합하도록 보강 작업을 완료하여 주시기 바랍니다.

* 업무 분야별 주요 작업
  • 프로그래머는 레이블 무결성과 함께 로그인 등 서식이 자바스크립트 없이 키보드로만 운용 가능한지 확인.
  • 콘텐츠 디자이너는 플래시, 동영상, 이미지(조직도, 약도, 도표) 등 주요 콘텐츠 위주로 보완.
  • 코더는 기능적인 부분 동작(자바스크립트)의 접근성 여부와 함께 작업자들이 놓친 부분이 없는지 점검.
  • 기획자는 작업을 원할하게 할 수 있도록 미흡한 콘텐츠 찾아주시고 10월 10일 전에 완료 여부를 반드시 체크해주시기 바랍니다.

품질마크 사전설문지 작성 요령

품질마크 사전설문지 작성 확인 및 수정해달라는 것은
클라이언트 의도가 품질마크 획득인가 사이트 점검인가 파악하여
유리한 쪽으로의 작성을 지원해달라는 겁니다.

===== 웹 접근성 품질마크 사전설문지는 =====
심사자의 편의를 위해서 제공하는 것뿐입니다.
얼마나 정확하게 잘 작성했는가가 심사 대상이 아닙니다.
그러므로..
웹 접근성 점검이 목적이라면 심사받고자 하는 콘텐츠 위주로 작성하시고,
품질마크 획득이 목적이라면 접근성이 우수한 콘텐츠만 적으시면 됩니다.

===== 수정되었으면 하는 부분 =====

□ 사이트 이용현황

==> 접근성이 우수한 웹페이지만 나열하셔도 됩니다.

□ 로그인 서비스 영역

==> 웹사이트의 내용은 대부분 로그인 없이 볼 수 있으므로 ② 아니오 ( ○ ) 로 하셔도 됩니다.
로그인이 필요한 my○○ 서비스는
접근성 문제가 있는 커뮤니티(동호회, 홈페이지마법사, 웹메일 등)로의 링크를 제공하므로 
심사 대상에 포함시키는건 좋지 않습니다.

□ 멀티미디어 및 기타파일 형식

==> 모든 콘텐츠를 다 기록하실 필요는 없습니다. (원본 양식에도 이렇게 되어 있네요.)
대표적인(웹 접근성이 우수한) 것 1개씩만 적으셔도 됩니다.

□ 품질마크 범위

==> 제외 서비스(부가 사이트)는
신청 사이트의 하위 폴더, 하위 도메인 중 웹 접근성이 부족한 것들을 모두 기재하시면 됩니다.
신청 사이트 외의 부가 사이트는 모두 제외하는게 품질마크 획득에 도움이 되며,
너무 많을 경우 신청 사이트 외는 모두 제외시킨다고 명시해주시면 됩니다.
