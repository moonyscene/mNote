    CSS(id, class) Naming Rule

    id, class Naming Rule

    id, class Naming Rule
    1. 영문 (a…z A…Z), 숫자 (0…9), 언더라인 (_), 하이픈 (-)만 사용한다.
      HTML 에서는 콜론(:)과 마침표(.) 도 가능하지만, CSS, JavaScript 에서는 오류 발생한다.
      <div id="AaZz0-9_:."></div> <!-- HTML(O) CSS(X) JavaScript(X) -->
    2. 첫 글자는 영문 소문자 (a…z)만 사용한다.
      1. 첫 글자로 숫자 (0…9) 가 올 수 없다.
      2. 첫 글자로 언더라인 (_) 또는 하이픈 (-) 이 올 수 있지만 이때 숫자가 뒤따를 수 없다.
    3. 카멜표기가 필요할 때만 대문자 사용한다.
    4. 철자를 구분해야할 경우, 1 (number), _ (Underline), A (CamelCase), - (Hyphen) 순서로 사용한다.
      - 를 사용할 때는 HTML 주석문에 - 를 적을 때 문법 오류가 생기는지 확인해봐라!
    5. 단위 콘텐츠 래퍼 이름은 의미있는 단어와 콘텐츠 코드 순으로 일련번호를 사용한다.
      예) .menu1
    6. 단위 콘텐츠 래퍼 이름을 활용한 하위 네이밍은 단순하게 한다.
      예) .menu1c, #menu1c1, #menu1c2
    7. 단위 콘텐츠 내부 요소들의 이름은 영문 소문자 1개 (아니면 2개)와 일련번호를 사용한다.
      예.기본) .m1, .m2, .m3
      예.단순) m.m1, m.m2, m.m3, g.g1, g.g2, g.g3
      예.복합) group.g1, group.g2, group.g3
      기반 스타일 (base.css 등) 에 정의해둔 속성값 단독 클래스 (.dpn, .tac, ..) 와의 출동을 피한다.
      ※) 속성값 단독 클래스는 3자이상으로 만들었으나, 오래전 만든 플로트 속성 ( .fl, .fr, .fn, ) 은 2자이고, 호환을 위해 계속 존재한다.
    8. 단위 콘텐츠 내부 요소들을 간단히 그룹 지을 때는 일련번호를 사용하지 않아 충돌을 피한다.
      단위 콘텐츠 내부 요소 이름을 유의미하게 만들려거든, 단위 콘텐츠 래퍼 이름과의 충돌을 피하기 위해 일련번호를 사용하지 않는다.
      예) .menu
    9. 단위 콘텐츠 래퍼는 class 로 정의 하여 여러번 사용할 수 있게 한다.
    10. id 는 a 요소 href 속성으로 링크 걸 때, 공통UI, 메인 콘텐츠 등 우선순위를 둘 때 등으로 국한시킨다.
      예) #bbs1c1, #menu1
    11. 목록은 중첩 고려해서.. class 명을 부여한다. .lst1, .li1, .dl1, .dt1, .dd1

    Efficient CSS Selector

    @m.Task) #s1 ul li{} 와 #s1 li{} 중 어느 것이 더 빠른가? )) CSS Rules는 적을수록 빠르다.

    /Reference/[book]자바스크립트 테스트와 디버깅.html#efficient-css - CSS 선택자 개선 전략

    CSS 선택자 개선 전략 (http://tinyurl.com/efficient-css/)

    1. 미사용 선택자 삭제. CSS 파일 크기 작아지고, 파싱 성능이 개선된다.
    2. 자손 보다는 자식 선택자를 자식 선택자 보다는 클래스나 아이디를 사용한다. 예) ol li a {} ▷ .ol-li-a {}
      하지만 모든 엘리멘트에 클래스나 아이디를 넣는 것은 불가능하기 때문에 적절히 이용한다.
    3. 불필요한 선택자 삭제. 예) div#some {} ▷ #some {} [m] 주의! div.some{} 는 권장한다.
    4. [IE7]이하에서 :hover 를 링크 외에는 사용안함. 사용하면 매우 느리다. 대안) JavaScript 로 onmouseover 구현.
    5. 공통(Universal) 선택자 * 를 key 선택자로 사용안함.
    6. 브라우저 렌더링 엔진은 CSS 선택자를 가장 오른쪽에서부터 왼쪽 방향으로 선택자를 탐색한 후 적합한 엘리먼트에 스타일을 적용한다.
      이때 가장 우측에 있는 선택자를 키(key) 선택자라고 한다.
      이 키 선택자가 * 와 같은 공통 선택자라면 매칭되는 엘리먼트가 많아서 성능이 느려진다.
      따라서 키 선택자는 특정 클래스와 같이 구체적으로 적는 것이 중요하다.

    CSS Coding Convention

    스타일시트 코딩은 선택자{속성:값;} 형식으로 1줄씩 작성한다. 작성하거나 속성:값; 별로 개행하는 방식을 혼용한다.
    더 나은 방법이라고 정해진건 없다. 핵심은 본인의 가독성!
    세로라인이 짧은거, 가로라인이 짧은거 중에.. 잘 보이는 거를 선택하면 된다.
    본인 경험에 의하면.. 반응형 코드처럼 비교가 빈번한 거만 개행하니.. 보기 편하였다.
    selector1{background:#fff;line-height:1.5;} selector2{background:#fff;line-height:1.5;}
    복수 Selector를 지정할 경우 각 셀렉터를 콤마(,)후 개행 한다.
    div, td, .selector{ } 단, 필요 시 한 줄 작성을 허용한다. reset 용 css, 의사 선택자 등.. a:visited:hover, a:visited:active, a:visited:focus{color:#09c;text-decoration:underline;}
    Style은 class 형태로 사용한다. (인라인 스타일시트 속성 style="" 형태는 피한다.)
    <div style="text-align: center;"> → (X) <div class="tac"> → (O) - 아래 스타일시트를 미리 정의해둔다. <style type="text/css"> .tac{text-align:center;} </style>
    한글 글꼴 선언 시 폰트 이름을 '한글',english 모두 기입한다.
    font-family:dotum; → (X) font-family:'돋움',dotum; → (O)
    font-size 는 pt, px 대신 em단위로 통일한다.
    font-size:11pt; → (X) font-size:11px; → (X) font-size:.92em; → (O) 단, body 태그에는 IE의 em 단위 해석 오류가 있어 % 단위를 사용 body{font-size:75%;}
    ※ 외부 스타일시트 <link rel="stylesheet" type="text/css" href="style.css" /> 형태로 사용하면 CSS 파일이 캐싱되어 웹 사이트 성능에 도움이 된다.

    하지만, 이렇게 하면 간혹 외부 스타일시트 파일을 불러오지 못하여 디자인이 깨지는 현상이 발생할 수도 있다.
    이런 현상이 지속되면.. HTML 페이지의 head 요소에 내부 스타일시트 <style type="text/css"> ~ </style> 형태로 직접 코딩하여 오류를 방지할 수 있다.
