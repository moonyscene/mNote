[A] 01. AUTO_REFRESH & REDIRECTION 자동갱신과 이동
만일 meta 요소가 "refresh" 값을 가진 http-equiv 속성과 함께 있으면,
content 속성에 의해 명시된 URI가 현재 리소스의 URI가 아니면 불합격(FAIL),
그렇지 않으면 주의(warn).

만일 Refresh HTTP 헤더가 있다면,
헤더 value 안에 명시된 URI 가 현재 resource의 URI가 아니면 불합격(FAIL),
그렇지 않으면 주의(warn).
<meta http-equiv="refresh" content="0;url=/" />
[A] 02. CACHING 캐싱
HTTP response가 Expires, Cache-Control헤더 둘 다 포함하지 않고
meta http-equiv 요소가 없으며, 이 헤더들을 참조하면, 불합격(FAIL)
다른 적절한 헤더에 명시되었을 것으로 가정되는 meta content 속성의 값을 이용하
여 테스트를 계속 수행하며, 주의(warn)

만일 Cache-Control HTTP 헤더가 있고 "no-cache" 나 "max-age=0" 값을 가지면 주의(warn)

만일 Pragma HTTP 헤더가 있고 "no-cache" 값을 가지면 주의(warn)

만일 Expires와 Date HTTP 헤더가 있고 Expires 헤더가 Date 헤더가 나타내는 날짜보다 더 늦지 않은 날짜를 나타낼 경우 주의(warn)

만일 어떤 cache관련 헤더든 유효하지 않은 값을 가지면 주의(warn)

만일 HTTP 응답이 Last-Modified 헤더를 가지면
Last-Modified 헤더의 값과 일치하는 값을 가진 If-Modified-Since헤더를 더하여, 똑같은 URI을 다시 요청하고
HTTP 응답이 Last-Modified 헤더를 가지고 그 값이 똑같고,
HTTP 응답 상태 코드가 304가 아니면 주의(warn)

HTTP 응답이 ETag 헤더를 가지면
ETag 헤더의 값과 일치하는 값을 가진 If-None-Match헤더를 더하여, 똑 같은 URI 을 다시 요청하고
HTTP 응답이 ETag헤더를 가지고 그 값이 똑같고,
HTTP 응답 상태 코드가 304가 아니면 주의(warn)

[A] 12. MINIMIZE 최소화
주의 : 스크립트와 CSS 내의 관계 없는 공백 문자는 이 테스트에서 고려하지 않는다. 그러한 확장에 대해서는 이 규격의 다음 개정 시에 고려할 수 있을 것이다. 
pre, style, script 요소 또는 XML코멘트 밖에 있는 하나 이상의 공백의 문자열에서
공백 문자(2.4.10 공백 문자 참조)의 수를 계산(맨 처음은 세지 말고)
이 수에 XML 코멘트를 이루는 문자들의 수를 더한다. 이 총계는 이 문서내의 관계없
는 문자들의 수이다.
문서 내의 총 문자 수를 계산
관계없는 문자들의 수가 문서의 총 문자의 개수의 10%를 초과하면 주의(warn)
관계없는 문자들의 수가 문서의 총 문자의 개수의 25%를 초과하면 불합격(FAIL)

모바일 콘텐츠 제작하는 방법

홈페이지를 어떻게 작성할 것인가
유선과 무선 환경에 맞게 다른 페이지로 구성
모바일OK를 One Web 을 지향하지만..
비표준은 표준화 방식으로 개편
액티브X, 플래시를 Javascript, CSS 를 이용하여 개편..
작업 효율성을 고려
기존 사이트를 이용하여 구성할 경우 최대한 손실이 없도록 수정하고 스타일시트 등을 모듈화..
데이터 크기를 최소화
50Kbye 로 제한.. 이미지 개수 및 용량을 최소화.. 불필요한 태그 및 주석을 삭제..
해상도는 시험인증 기준이 아니다
그러나 UI, UX 측면에서 One 모바일 웹사이트를 구축하여 다양한 해상도 단말에 최적의 서비스를 제공..
효과적인 모바일OK 사이트 구현

스마트폰, Touch UI 등의 향상된 Native UX 제공으로 높아진 사용자 UX 요구에 부응할 수 있는 전략이 필요하다.

  • DDR 조회를 통해 단말에 최적화된 콘텐츠를 제공하도록 한다.
  • Pivot (세로 또는 가로 보기) 등의 부가적인 정보 활용하여 최적화된 사이트를 구성한다.
최적의 레이아웃 전략

가독성이 향상되어 이용자가 보다 쉽고 빠르게 내용을 파악하고 이해할 수 있다.

콘텐츠 성격에 따른 레이아웃
  • 아이콘, 글로벌 내비게이션 메뉴바, 게시판 목록 형태 등..
단말 성능에 따른 레이아웃
  • 액정크기 해상도 고려
사용자 조작성에 따른 레이아웃
  • 터치 입력 방식에 적절한 레이아웃
  • 링크는 사이의 간격을 적당히 띄어 주어.. 의도하지 않은 이동을 방지하도록 한다.
  • font-size 를 단말의 해상도와 브라우저의 종류에 따라서 적절히 조절
  • 글로벌 및 로컬 내비게이션을 제공하여 사용자의 링크에 대한 신속한 접근 및 그 페이지의 현재 위치를 쉽게 파악할 수 있도록 한다.
모바일 웹 디자인할 때 주의사항
  • 목록을 간략하게 구성하자
  • 여백을 충분히 확보하자
  • 이미지는 최소화하자
  • 별도의 도메인 대시 서브 도메인을 사용하자.
    .mobi 도메인 보다는 http://.site.com http://mobile.site.com http://site.com/mobile 형식을 많이 사용한다.
  • 우선 순위 - 방문자의 시선을 중심으로 구성
|TIP| 모바일 디자인을 위한 고려 사항
  • XHTML로 페이지 내용을 구조화시키고 CSS를 이용하여 디자인을 체계화하자.
  • 꾸밈보다는 콘텐츠 전달을 우선으로 생각하자.
  • img 태그의 alt 속성에 이미지 설명글을 표시하자.
  • 가능한 콘텐츠가 화면을 넘치지 않게 구성하자.
  • 여백을 너무 많이 설정하지 않도록 하자.
  • 모바일 액정 차이로 인한 색상 차이를 염두에 두자.
