    1. 안드로이드 이벤트 투과 문제 ※ http://jindo.dev.naver.com/jindo_home/doc/preventClickEventForAndroid.docx
      (증상) Android 4.0.x 버전 이하 Default Browser에서는 레이어를 띄울 경우, 이벤트가 투과 되어 레이어 뒷면에까지 이벤트가 전달되는 버그가 있다.
      이 문제는 원천적으로 해결할 수 는 없다. 하지만, 다음과 같은 접근 방식으로 문제를 해결할 수 있다.
      1. (해결1) 상위 레이어에 백그라운드 컬러를 주기 다만, 모든 단말기에서 처리할 수 있는 것은 아니고, 4.0.x버전의 LG단말기에서는 이러한 문제를 해결 할 수 있기 때문에 궁극적인 해결책은 아니다.
      2. (해결2) 레이어 뒷면의 엘리먼트 숨기기 레이어에서 투과될 수 있는 경우를 원천적으로 제거하는 방식으로, 하위 레이어에 존재하는 클릭 될 수 있는 엘리먼트를 제거하는 방식이다. 위 그림의 버튼이나 하위 레이어를 display:none 한다.
      3. touchstart에서 이벤트 막는 방법 이 방식은 상위 레이어에서 발생하는 모든 이벤트를 막고, 상위 레이어에서 투과되어야 할 엘리먼트에 대해서는 이벤트를 막지 않는 방식이다. 상위 레이어에서 발생하는 이벤트를 막기 위해서는 touchstart 에서 이벤트를 막아야 한다. 또한, 상위 엘리먼트의 버튼도 해당 액션 처리 후 이벤트를 막아야 한다. 아래는 class가 “pass” 라는 엘리먼트를 제외하고는 모든 이벤트를 touchstart에서 막는 코드이다. <button id="btn" class='pass'>버튼</button> jindo.$Fn(function(we) { console.log("버튼 선택"); we.stop(jindo.$Event.CANCEL_ALL); }, this).attach("btn", "touchstart"); jindo.$Fn(function(we) { var wel = jindo.$Element(we.element); if(!wel.hasClass("pass")) { we.stop(jindo.$Event.CANCEL_ALL); } }, this).attach("상위레이어", "touchstart"); 단, 안드로이드 2.x 대에서는 이벤트는 막히지만, 하이라이트나 롱 탭은 발생한다.
