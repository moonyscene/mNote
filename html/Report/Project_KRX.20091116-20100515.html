>신규 콘텐츠 CSS는 무엇을 참조해야 하고, 어디에 추가해야 하는지?
1.사전에 ~/css/default.css 에 선언된 CSS 를 숙지한다.
2.서브 페이지에 신규 콘텐츠 추가시 먼저 서브+본문 콘텐츠 마크업 템플릿 (~/css/default.css) 에 정의된 유형의 콘텐츠인지 확인한다.
3.없다면, ~/css/업무코드/default.css 에 정의된 유형의 콘텐츠인지 확인한다.
4.정의된 적이 없는 유형이고 현재 사이트에서만 사용할거면 ~/css/업무코드/default.css 에서 /* body_content ========== */ 아래의 마지막에 추가한다.
단, 특별한 경우가 아니면 class 선택자를 사용한다. 
5.여러 사이트에 공통으로 사용할 CSS 추가시 ~/css/default.css 의 /* Content ========== */ 와 /* 본문인쇄용 */ 사이의 CSS 그룹에 추가한다.
CSS 그룹은 구조 혹은 표현별로 유사한 것들을 모아둔 것으로서, /* 제목 */, /* 목록 */, /* 상자 */, /* 표 */, /* 탭 */ 등이 있다.
CSS 그룹이 없다면 /* 본문인쇄용 */ 바로 위 혹은 적당한 위치에 추가한다.

== 이클립스 사용법 ==
== Template 폴더 로컬서버와 SVN서버 동기화하기
(JDK+Tomcat+Eclipse 설치 세팅 완료 후)
* 이클립스 실행
* 이클립스 초기 화면 우측 아래 영역의 탭 Servers 를 클릭하여 활성시킨다.
* 탭 우측의 ▶(Start the Server) 를 클릭하여 Apache Tomcat 웹서버를 실행한다.
* 좌측 Pakage Explor 에서
* template 폴더로 가서 F5를 눌러 새로고침한다.(이클립스 실행 후 html 소스 수정된 내용 반영)
* template 폴더 우클릭 [Team]-[Synchronize with Repository]
* 좌측 Pakage Explor 탭 대신 Synchronize  탭을 보여준다.
* 로컬서버와 SVN서버의 내용을 비교하여 아이콘으로 동기화시켜야 할 것들을 보여준다.
- 아이콘 ←(파란색) SVN 에서 로컬로 받을 것 [우클릭하여 update]
　아이콘 ←(파란색) 안에 - 표시는 로컬에서 삭제된 파일을 SVN에서도 삭제할 파일 또는 폴더
　아이콘 ←(파란색) 안에 + 표시는 로컬에서 추가된 파일을 SVN에서도 추가할 파일 또는 폴더
- 아이콘 →(회색) 로컬에서 SVN으로 올릴 것 [우클릭하여 commit]
- 아이콘 ↔(붉은색) 같은 파일이 로컬과 SVN 에서 모두 수정되었을 때.
　파일 소스를 비교하여 다른 부분을 확인.
　SVN에 올린 최종 작업자를 찾아서 상호 확인 후 상황에 맞게 처리한다.
　소스 코드안에서 똑 같은 부분이 상호 수정된 경우가 아니라면 update, commit 하면 된다.
　소스 코드안에서 똑 같은 부분이 상호 수정된 경우는 로컬 파일 백업 후 update(로컬←SVN)한 후 비교 수정한다.
* 동기화 완료 후 Pakage Explor 로 돌아갈 때는 우측 상단 탭(Team Sync..) - Java 선택
== 유의사항
* 동기화 전에 SVN 해당 폴더 압축 백업한다.( common , image, template 와 해당 사이트 폴더에서 본인이 작업한 html 폴더, index.html 등을 묶어서)
* 일반적으로 update 완료 후 commit 은 실행하면 된다.
* update 와 commit 은 하나씩 혹은 여러개 선택하여 한꺼번에 할 수 있다.
