한국형 웹 콘텐츠 접근성 지침 (KWCAG 1.0) 13개 항목

  • 이전
  • 목차
  • 다음
  • 1. 텍스트 아닌 콘텐츠의 인식
    • KADO-WAH 준수율 90%이상
    • 대체 텍스트 내용 적절하게 제공 (텍스트 내용이 이미지 내용 정보를 모두 전달)
    • 정보가 있는 배경 이미지의 대체 텍스트 제공 (CSS 제거 후 확인)
    • ※ FF 에서 [Web Developer]-[Images]-[Display Alt Attributes] 기능으로 일부 확인할 수 있으나, 코드 수동 평가 모두 만족해야 함.
  • 2. 멀티미디어 매체의 인식
    • 동영상 자막 또는 원고 제공 (요약 정보만 제공하면 안됨)
  • 3. 콘텐츠의 시각적 명료성
    • 명암이나 패턴으로 콘텐츠 구분 (흑백 출력 확인)
    • 색상 대비 Fujitsu ColorSelector 적합 판정 ♥객관적평가?
    • 폰트 12px 이상 ♥객관적평가?
    • 상대폰트 사용 (IE6 [보기]-[텍스트크기]) ♥애매모호?
  • 4. 이미지 맵 기법 사용 제한
    • 서버측 이미지맵 대체콘텐츠 (GIS는 예외).
  • 5. 프레임의 사용 제한
    • 프레임 title 속성 제공 (KADO-Wah 자동 평가 준수율 95% 이상)
    • 4개 이상 프레임 사용 제한 ♥객관적평가?
  • 6. 깜박거리는 객체 사용 제한
    • 메인 깜박이는 콘텐츠 제한 (감광성 간질 자동평가도구?) ♥애매모호?
    • 사전경고, 대체 페이지 링크 제공
    • ※ 그동안 평가에서.. 플래시 움직임은 깜박임이 아닌 것으로 판단.
  • 7. 키보드로만 운용 가능
    • 탭 키로 처음부터 끝까지 이동이 되는지 검사
    • 탭, 엔터 키로 사용에 무리가 없는 경우에는 준수
    • <select> 요소에 이동 버튼 (submit 기능) 제공
    • ※ 그동안 평가에서.. focus 로 상황이 바뀌어도 감점하지 않음. ♥애매모호?
  • 8. 반복 내비게이션 링크
    • CSS 제거 후 건너뛰기 링크 확인
  • 9. 반응시간의 조절기능
    • 자동 갱신 콘텐츠는 사용자가 이를 제어하고 (지나간)콘텐츠를 확인할 수 있는 수단을 제공
    • 포커스시 멈춤 + 이전, 다음 보기 기능 제공
    • 새 창(팝업창) 사전에 알림
    • ※ 그동안 평가에서..
      메인 페이지 팝업창 1개만 허용하였지만, 좀 더 고려중~ ♥객관적평가?.. 팝업창 띄우지 말자!
  • 10. 데이터 테이블 구성
    • 제목을 <caption>, <h1>~<h6> 중 하나라도 제공하는지 확인
    • <thead>, <tbody>, <th> 태그, summary="" 속성 사용 여부 확인
    • ※ 그동안 평가에서.. scope="col" scope="row", abbr 속성은 평가하지 않음. ♥제외/삭제?
  • 11. 페이지의 논리적 구성
    • <title>, <h1>~<h6>, <p>, <ul>,<ol>,<dl> 등 적절한 HTML 사용 확인
    • URL을 고정하기 위한 빈 프레임 제거
    • CSS 를 이용하여 구조와 표현(디자인)을 분리
    • 논리적 순서로 구성되어 있는지 확인 (CSS 제거 후 배치 확인)
    • 레이아웃 테이블에 <caption>, <thead>, <tbody>, <tfoot>, <th> 있으면 제거
    • 링크의 텍스트 혹은 title="부가설명" 을 명확하게 제공
    • ※ 그동안 평가에서.. 디자인 요소 마크업 사용시 지적한 경우는 아직 발견 못함.
      디자인 요소 마크업 : <br>, <center>, <font color="" size="" face="">, align, bgcolor, valign, border, ..
  • 12. 온라인 서식 구성
    • 서식에 <label> 제공
    • 로그인, 검색 등 키보드 운용(입력 후 그 전 서식 수정 포함) 확인 [지침07에서 체크]
  • 13. 신기술의 사용
    • 플래시 자체적인 접근성 준수 (이동 순서, 대체텍스트)
    • 플래시 등 대체콘텐츠 제공 [지침01에서 체크]
    • javascript 없이 링크, 서식, 버튼 작동
    • javascript 없이 동일한 내용 제공 (js에서css참조)
    • 뷰어 프로그램 링크 제공
    • ※ 공인인증서, 보안결재는 접근성 보장 애플리케이션이 나오기 전에는 예외로 인정
