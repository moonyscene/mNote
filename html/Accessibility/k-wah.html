※ K-WAH 3.0 평가 환경 설정

자동 평가 시간이 너무 오래 걸리므로 보강 작업시에는 빨리 진행할 수 있도록 설정을 조정하더라도..
** 최종 평가시에는 아래와 같이 설정합니다.

  • 평가방법 : 원격 시스템 평가
  • 최대 깊이 제한 : 105 (기본값 5)
  • 최대 페이지 수 제한 : 1000 (기본값 100)
  • 다운로드 페이지 연결 시간 제한 : 10초 (기본값 10)
  • 게시판 반복 검사 제한 : 5 (기본값 5) - 1000 페이지 이상이거나 점검 작업중일 때는 1로 설정하자!
  • 검사항목 선택 : 인터넷 웹 콘텐츠 접근성 지침1.0
  • 검출항목 : 오류항목, 경고항목, 움직이는 이미지 평가여부, 플래시 평가여부 모두 체크
※ 정보화진흥원 제8회 웹 접근성 품질마크 전문가심사 보고서(2010.5)의 K-WAH 3.0 자동평가 결과(참고자료)는 심사 대상 20개 페이지만 설정하여 검사한 결과를 첨부하였다.

S/W 툴 자체 결함

  • 서식에서 id="문자열" 과 비교하여 for="다르더라도문자열포함하면" 오류로 검출되지 않는다.
    <label for="xAAA2">선택옵션</label><select id="AAA">..중략..</select> 또한, title="" 속성이 있으면 for 연결이 올바르지 않더라도 오류로 검출되지 않는다.
    <label for="xABC2">선택옵션</label><select id="AAA" title="선택옵션">..중략..</select> id 와 for 연결이 올바른지 소스 코드 확인이 반드시 필요하다!
  • public.wonju.go.kr 돌려보니 오류가 있는데도 종합보고서 그래프에는 미준수율0% 로 나오는 경우도 있음.
    해결책) 99.x% 준수율일 경우 그렇게 나오므로, 상세 보고서를 반드시 확인하자.
  • 평가마법사 웹 사이트 주소의 기본문서가 다른 페이지로 즉시 Redirect 하면 자동평가 안되는 경우도 있음.
    예) lifelong.jinju.go.kr 여기는 자동평가 안되다가 한달 뒤에 해보니 됩니다.
    해결책) 몰라.
  • 평가마법사 웹 사이트 주소가 http://yc.go.kr/main/ 일 경우.. 파일보고서의 파일명에 / 가 중복됨
    예) http://yc.go.kr/main//program/publicboard/inBoardDoc.asp!
    해결책) yc.go.kr/main 마지막에 / 안나오게 이렇게만 기입하자.
  • 파일보고서의 파일명에 파라미터 값을 넘기는 ? 를 ! 로 잘못 표시함.
    예) http://www.tongyeong.go.kr//program/incsilboard/lstSilBoard.asp!Bod01Code=1

K-WAH 3.0 검사항목 중 유의해야할 것들

구분 비고. K-WAH 3.0 Rule 설명
Rule No.17 현재 대부분의 브라우저는 onclick 이벤트에 대해서 onkeypress를 동시에 지원해주므로 현실적으로는 onkeypress를 명시할 필요가 없습니다.
(오류) <a>, <button>, <input> 요소의 마우스 이벤트에 대한 키보드 이벤트가 존재하지 않을 경우 보고 됩니다.
- onmouseover에 대한 대응 키보드 이벤트 onfocus 사용 안된 경우
- onmouseout에 대한 대응 키보드 이벤트 onblur 사용 안된 경우
- onmouseup에 대한 대응 키보드 이벤트 onkeyup 사용 안 된 경우
- onmousedown에 대한 대응 키보드 이벤트 onkeydown 사용 안된 경우
- ondbclick이 사용된 경우
object 요소의 대체콘텐츠가 태그로 시작되지 않으면 오류로 표시 확인필요
웹사이트 주소에 port 가 사용되거나 특정 ActiveX 사용시 평가 마법사 안되는 s/w 오류 확인필요
Rule No.4 (오류) <applet> 요소에 alt 속성이 없을 경우 보고됩니다.
Rule No.28 (경고) <table>을 사용한 경우 자식요소를 포함하지 않는 최하위 <table>만을 검출한 다음 summary 속성이 없거나 <caption /> 요소가 발견되지 않으면 보고 됩니다.
Rule No.35 (오류) 다음 세가지 조건 중 하나라도 만족하지 못하면 보고됩니다.
<input>, <textarea>, <select> 요소의 id와 동일한 값을 지닌 <label for="" />요소가 없을 경우
<input>, <textarea>, <select> 요소의 title이 없을 경우
<input>, <textarea>, <select> 요소의 자식요소가 제공되지 않은 경우
(* <input> 요소의 type 이 submit, image, button 인 컨트롤은 제외)
Rule No.37 (경고) <object /> 요소가 <param /> 이외 다른 형식의 HTML 요소나 텍스트를 자식으로 포함하지 않은 경우 보고 됩니다.
Rule No.38 (오류) <a> 요소의 href 속성값에 "javascript :" 슈도 프로토콜(pseudo-protocol)을 사용한 경우 보고 됩니다.
또는 <a> 요소의 href에 값이 없거나 href=“#” 값만 존재하는 경우 보고 됩니다.
Rule No.39 (오류) <form> 요소에 action 속성이 비어있을 경우 보고 됩니다. 또는 <form> 요소에 action속성값이 " javascript:"로 시작 되면 보고 됩니다.
Rule No.40 (오류) <form>요소에 <input type="submit">이나 <input type="image">, <button type="submit">이 없는 경우 보고 됩니다.
Rule No.41 (오류) 실제로는 다른 링크인데 <a> 요소의 title 속성 값과 합친 링크의 텍스트가 동일하면 보고됩니다.
또는 실제로는 다른 링크인데 <a> 요소의 하위 요소인 <img>의 alt 속성값이 동일하면 보고됩니다.
