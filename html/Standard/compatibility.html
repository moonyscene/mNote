    호환성 진단

    전자정부서비스 호환성 준수지침 [시행 2012.10.10] [행정안전부고시 제2012-44호, 2012.10.10, 일부개정] - 국가법령정보센터 -
    동작, 레이아웃, 플러그인 호환성
    크로스 브라우징 테스트 : 3종 웹브라우저 ( IE, Chrome, Firefox, Safari, Opera 에서 선택)

    실제 진단 결과

    웹 표준 문법 준수(W3C Markup Validator Errors) (2009 행안부 진단)

    1. 특수문자(&)를 Name Code(&amp;) 형태로 사용하지 않은 경우.
    2. HTML4.01 에서 종료 태그가 없는 홀로 쓰이는 태그에 종료 표시를 해준 경우. <meta /> <link /> <base /> <col /> <area /> <frame />
    3. 비표준 속성 allowtransparency="true" 를 사용한 경우. (웹표준 아니지만, 대부분의 브라우저에서 지원.)
    4. img 요소의 필수 속성 alt 가 없는 경우.
    5. script 요소의 필수 속성 type 이 없는 경우.
    6. 요소에 없는 속성을 사용한 경우. onfocus, onblur, target 등..
    7. 같은 속성을 중복해서 사용한 경우. tabindex="0" 등..
    8. </a></li></a></li> 와 같이 잘 구성(well-formed)되어 있지 않은 경우.
    9. <ul> 하위에 <li> 없이 다른 요소를 사용한 경우.
    10. <a href="#"><a href="#"></a></a> 와 같이 중첩 앵커를 사용한 경우.
    11. 종료 태그 </div> </span> 등이 모자라거나 남는 경우.
    12. 태그의 속성 값들에 따옴표(" 또는 ')를 사용하지 않은 경우.
    13. ""(따옴표)를 중복으로 코딩한 경우.
    14. 주석 안에 불필요한 -(dash)가 있는 경우. <!-- <!-- -->
    15. document.write(); 함수 안의 마크업 코드가 문법 오류로 해석된 경우.
    16. <html></html> 외부에 자바스크립트 코드 등을 사용한 경우.
    17. 인라인 요소 안에 블럭 요소가 포함된 경우.

    호환성 보장

    메타 태그

    개발자는 메타 태그의 콘텐츠 특성을 사용하여 웹 페이지에 대해 콘텐츠가 렌더링되는 모드를 지정할 수 있습니다. Internet Explorer 8 표준 렌더링 모드를 사용하여 콘텐츠가 표시되게 하려면 IE=EmulateIE8 또는 IE=8 값을 지정합니다. Internet Explorer 9 표준 렌더링 모드를 사용하여 콘텐츠가 표시되게 하려면 IE=EmulateIE9 또는 IE=9 값을 지정합니다. Internet Explorer 9에서 Internet Explorer 7 렌더링 동작을 사용하여 콘텐츠를 표시하려면 IE=EmulateIE7 또는 IE=7을 지정합니다. Internet Explorer 5.5 또는 쿼크를 사용하여 콘텐츠가 표시되게 하려면 IE=5 값을 지정합니다.

    <meta http-equiv="X-UA-Compatible" content="IE=Edge" /> <meta http-equiv="X-UA-Compatible" content="IE=edge,chrome=1" /> [펌] 2라인 코드는 크롬과 IE 모두 설치된 PC 에서는 크롬의 렌더링 엔진을 차용해서 화면에 뿌려주지만, 아닌 경우에는 그냥 IE 모습 그대로.. ㅠ. 그래도, 호환성보기 옵션을 선택할 수 없게 된다는 것 자체에 의미를 두어야.. ^^.
    head 요소 안에서 위치는 <meta charset="utf-8" /> 바로 다음이 좋을 듯하다.
    그 앞에 넣어도 되지만, link 요소, script 요소 다음에 오면 동작 안한다.
    또한, HTML5 유효성 통과 못한다. 20140730 현재.
    유효성 통과 위해.. 조건주석문, 스크립트 안에 넣는 방법 등을 사용해보니.. 유효성은 통과되나, 동작하지를 않는다.
    [m] ",chrome=1" 없이 사용하면 HTML5 문법 유효성 통과한다. <meta http-equiv="X-UA-Compatible" content="IE=Edge" />

    호환성 및 X-UA-Compatible 헤더에 대한 자세한 내용은 문서 호환성 정의를 참조하십시오.

    HTTP 헤더

    웹 사이트에는 수천 또는 수만 개의 개별 페이지가 포함될 수 있으므로 각 문서에서 이 값을 설정하는 것은 비효율적입니다. 모든 페이지나 폴더별로 선택한 페이지 컬렉션에 대해 메타 태그를 설정하여 사이트가 작동하는 경우 서버 구성을 조정하고 HTTP 헤더에 X-UA-Compatible 메타데이터를 추가하는 것이 좋습니다.

    다음 표는 Internet Explorer 9의 문서 호환 모드에 대해 설명합니다.

    Internet Explorer 9의 문서 호환 모드
    콘텐츠 값 의미
    IE=5 "쿼크" 모드
    IE=7 항상 Internet Explorer 7 모드를 사용합니다.
    IE=EmulateIE7 사이트에 쿼크 DOCTYPE이 없는 경우 Internet Explorer 7 모드로 표시됩니다.
    IE=8 항상 Internet Explorer 8 모드를 사용합니다.
    IE=EmulateIE8 사이트에 쿼크 DOCTYPE이 없는 경우 Internet Explorer 8 모드로 표시됩니다.
    또한 이 태그를 사용하여 클라이언트 컴퓨터에서 호환성 보기를 무시하고 Internet Explorer 8 모드를 강제 적용합니다.
    IE=9 항상 Internet Explorer 9 모드를 사용합니다.
    IE=EmulateIE9 사이트에 쿼크 DOCTYPE이 없는 경우 Internet Explorer 9 모드로 표시됩니다.
    또한 이 태그를 사용하여 클라이언트 컴퓨터에서 호환성 보기를 무시하고 Internet Explorer 9 모드를 강제 적용합니다.
    IE=Edge 최신 모드로 표시합니다. Internet Explorer 9에서는 IE=9과 같습니다.
