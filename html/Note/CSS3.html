gradient [CR][SF][FF][OP][IE] 효과적다. 이미지가 더 쉽고 유연한 디자인 가능. background: #3EAF0E -webkit-gradient(linear, 0 0, 0 100%, from(#3EAF0E), to(#fff));
background: #3EAF0E -moz-linear-gradient(top, #3EAF0E, #fff);
filter:progid:DXImageTransform.Microsoft.gradient(startColorStr=#3EAF0E, endColorStr=#ffffff);
 속성 및 값 선언 방식 모두 다름.
webkit[SF,CR]은 (타입, X시작점 Y시작점, X종료점 Y종료점, 시작색, 종료색)
moz[FF]는 (시작점, 시작색, 종료색)
gradient 미지원 위해 기본 배경색을 지정 필요
transform [CR][SF][FF][OP][IE] 효과적다. 텍스트일 경우 적용해보자. 이미지 적용은 별로다. -o-transform: rotate(90deg); -o-transform-origin: 0 100%;
-moz-transform: rotate(90deg); -moz-transform-origin: 0 100%;
-webkit-transform: rotate(90deg); -webkit-transform-origin: 0 100%;
filter:progid:DXImageTransform.Microsoft.Matrix(M11=6.123031769111886e-17, M12=-1, M21=1, M22=6.123031769111886e-17, sizingmethod='auto expand');
 시계 방향으로 90도 회전시키는 코드.
translate, scale, rotate, skew(비틀기) 가능.
transform-origin 는 transform의 축 지정
기본 값 50%(X축) 50%(Y축)은 생략 가능.
IE는 사용법이 다소 복잡하다.
transition [CR][SF][FF][OP][IE] 효과적, IE지원까지 보류. a { padding: 10px 15px; margin: 0; -o-transition: .2s; -webkit-transition: .2s; -moz-transition: .2s; }
a:hover { padding:15px 20px; margin: -6px; opacity: 0.8; }
 transition 속성의 값으로는 어떤 속성을 몇 초간 진행할 것인지 정의한다.
어떤 속성을 transition 할 것인지 지정하지 않으면 기본 값 all이 지정되어 모든 속성을 transition 시킨다.
파이어폭스 브라우저는 3.7 버전부터 지원을 기대할 수 있다.
animation [CR][SF][FF][OP][IE] 필요한 곳이 있을까?, IE지원까지 보류. a:hover { -webkit-animation: webkitFire infinite 1s linear; }
@-webkit-keyframes 'webkitFire'{
from{ -webkit-transform: rotate(0deg); }
to{ -webkit-transform: rotate(360deg); }
}
 'webkitFire'라는 사용자 정의 애니메이션이 a:hover 상태에서 실행.
:hover 동안 일정한 속도로 1초에 한 바퀴씩 영원히 돈다.
