항상! 다음 작업자와 유지보수 편의를 고려하면서 작업해주시기 바랍니다.

  1. PSD 파일명에 버전과 작업자 이름 등을 표시해주세요!
    (갑.프로젝트명_)사이트명_웹문서명_(반응형구분)_20140320(최종본)_디자이너.psd
    • (예1) 기본 형태
      영천_문화관광_Sub(보현산천문대)_20101012_황진이.psd
    • (예2) 반응형, 디자이너 이관.
      남해군_열린군수실_Main_Desktop_성춘향_20140813_향단이.수정.psd
      남해군_열린군수실_Main_Tablet_성춘향_20140813_향단이.수정.psd
      남해군_열린군수실_Main_Smartphone_성춘향_20140813_향단이.수정.psd
    • (예3) 반응형. 중단점에 따른 구분 -- 가장 큰 화면은 _Sub@3_ 처럼 중단점을 넣지 않고, 기본 형태 _Sub_ 로 표시하였음.
      울회사.웹사이트.리뉴얼_Sub_1024~_20140813(최종본)_성춘향.psd
      울회사.웹사이트.리뉴얼_Sub@1_320~768_20140813(최종본)_성춘향.psd
      울회사.웹사이트.리뉴얼_Sub@2_768~1024_20140813(최종본)_성춘향.psd
      또한 중단점 표시는 웹 이미지 파일명을 만들 때 그대로 적용한다.
      menu2s.png
      menu2s@1.png
      menu2s@2.png
  2. "디자인 포토샵 파일(*.psd)", "콘텐츠 구조 엑셀 파일(*.xls)"이 확정된 최종본인지 확인하고 넘겨주세요.
  3. 온/오프 상태, 탭메뉴, 로그인 후 등 상황에 따라 달라지는 이미지는 +on +off 별로 Layer Set|Group 을 만들어 주십시오.
    Layer Set|Group 명명 방법은 [+이름] 을 권장합니다. [예] ex)TabOnOff_myshin.png [psd소스]
  4. 상단메뉴, 좌메뉴의 하위메뉴도 상황별로 디자인 추가 변경 있다면 별도 Layer Set|Group으로 만들어주세요.
  5. Layer Set|Group 만들 때 콘텐츠 구역별로 잘 묶어서 주시고, 가급적 레이어 이름을 남겨주세요.
    • 테이블 레이아웃은 디자인 위주로 자르지만, CSS 레이아웃은 콘텐츠별로 자르므로 Layer Set|Group 숨김, 보임을 자주하게 됩니다.
      레이어 정리가 안되어 있을 경우(나쁜예: head 와 body의 텍스트를 하나의 Layer Set|Group 안에 두는 경우) 커팅 작업이 곤란합니다.
    • 불릿은 bu 혹은 ㅇ, 라인은 line 혹은 ---- 등등 간단하게라도 붙이는 습관을 가져주세요.
      (시간 많이 안걸립니다. - 수정할 때, UI코딩시, 유지보수시 편리)
    • 필히, 전체 배경 디자인은 별도 Layer Set|Group[+bg]으로 구분하시고,
      영역별 Layer Set|Group 안에서도 배경[+bg]과 콘텐츠[+c] 로 크게 2개의 Layer Set|Group 으로 나눠주시면 배경만 커팅해야 할 경우 작업이 수월합니다.
  6. 연관된 레이어는 바로 인접한곳에 위치하도록 해주세요.
    나쁜예 : 텍스트를 꾸미는 불릿 레이어가 텍스트 레이어와 인접해있지 않고 사이에 다른 레이어들이 어지럽게 섞여있는 경우.
  7. 이미지 메뉴는 크기 또는 간격이 일정하게 제작해주세요.
  8. 온·오프(활성·비활성) 이미지 크기를 동일하게 만들어주세요. (가로형은 너비, 세로형은 높이가 필히 같아야 함)
  9. 그리드 계획에 맞춰 주세요. (가이드라인 사용시 1px씩 어긋나지 않게 정확하게 표시해주세요.)
    ※주의: 포토샵 작업시 확대 배율이 100% 의 배수가 아닐 경우 가이드가 1px 중간에 만들어지는 오류.
  10. 여러 페이지에 걸쳐서 같은 형태의 디자인일 경우 psd 파일 1개로 처리해주세요. (예) 정보행동 - 개발솔루션
    서브 페이지 UI(head, sidebar, foot 영역), 메인 페이지 디자인은 각각 psd 파일 1개로 해결할 수 있도록 해주세요.
    주메뉴, 푸터 등 메인과 서브 공통 템플릿 디자인은 서브 템플릿 디자인 psd 파일에 모두 완전하게 넣어주세요. (일반적으로 서브 코딩이 먼저 진행됨)
  11. 플래시 링크와 비플래시 링크가 가급적 겹치지 않도록 작업해주세요. (나쁜예) 통합전(구)마산시
  12. 서브 플래시 비쥬얼 위에 메뉴를 만들지 마세요. (나쁜예) 통합전(구)마산시
  13. 디자인시 색상 대비에 좀 더 신경을 써주세요.
    한국형 웹 콘텐츠 접근성 지침 2.0 부록 Ⅲ 한국형 웹 콘텐츠 접근성 지침 2.0 검사항목(22개) 사례
    ⑤ (텍스트 콘텐츠의 명도 대비) 텍스트 콘텐츠와 배경 간의 명도 대비는 4.5대 1 이상이어야 한다. 부분 참고
  14. PSD 파일이 너무 커서 컴퓨터가 버벅거립니다.(10M 20M이하를 원하는데, 20M가 넘는 경우도 많군요.)
    미사용 레이어를 삭제하고, 추후 수정이 필요없는 것들은 머지시켜서 용량 좀 줄여주시면 안될까요?
    (색)조정 레이어, 투명도 사용, 레이어 마스크, 폴더 등을 많이 사용하면 용량이 엄청 늘어납니다.
  15. 비쥬얼이 크면, 메인과 서브 전체 디자인 PSD 에는 대표 비쥬얼만 남겨두시고,
    서브페이지별 비쥬얼, 메인 비쥬얼 장면별 디자인은 별도 PSD 파일로 만들어주세요.
  16. 페이지 레아아웃 백그라운드에 그라데이션, 그림자가 없으면 안될까요?
    코딩시 쓸데 없는 코드들을 많이 생성시키고, 페이지 전체 렌더링 속도가 엄청 무거워집니다.
    그림자가 필요할 경우.. 광원, 광량을 일정하게 하여 전체적인 z좌표가 flat하게 해주세요.
  17. 이미지를 투명 png-24 파일로 저장해야 하는 경우가 없도록 디자인해주십시오. (나쁜예) 통합전(구)마산시
    IE6,7에서 버그가 있어서 비표준코드를 생성해야하고 코드가 복잡해집니다. 단, IE6,7 호환을 고려하지 않는 웹사이트에는 사용하셔도 됩니다.
    플래시 내비게이션은 만들지 않으니, 디자인 발상을 다르게 해주시기 바랍니다.
  18. 좌메뉴 등 이미지가 아닌 텍스트 정보는 포토샵에서 자간을 100의 배수(css에서 letter-spacing 1px의 배수)로만 지정해주세요.
    포토샵에서 Character 패널의 kerning 을 0 또는 Metrics 로 설정하세요. (Optical 은 사용하지 마세요.)
  19. 메뉴 영역, 제목 영역 등 모든 콘텐츠와 배경은 예상할 수 있는 모든 경우를(가장 짧은 것과 가장 긴 것을) 고려하여 디자인해주세요.
  20. 여러 사이즈(원본, 섬네일 등)로 서비스 되는(DB에 저장되는) 이미지는 가로 세로 비율을 맞춰주세요.
    (모니터 화면, 디카의 일반적인 사이즈인 4:3 비율을 권장합니다.)
    디자인하기 전에 기존 DB 이미지를 그대로 사용할 것인지, 수정해서 다시 등록할것인지 파악한 후에 디자인해주세요.
  21. 여러 페이지(서브와 메인 등)에서 같은 콘텐츠가 다른 위치에서 보여질 때 가능하면 디자인을 맞춰주세요. (나쁜예) 메인과 서브의 "금주의 추천농장"
  22. 외국어 디자인시 텍스트 콘텐츠는 해당 언어 OS 범용 폰트로 만들어주세요.
    국문: 돋움, 굴림, ..
    영문: Tahoma, Arial, Verdana, ..
    일문: "MS UI Gothic", "MS Gothic"
    중문: Simsun, SimHei, "MS Song"
  23. 웹사이트 화면에 기능이 필요한 경우, 디자인 전에 자바스크립트 개발자와 반드시 상의해주세요.
  24. 포토샵 텍스트 레이어의 정렬 속성을 디자인의 정렬과 동일하게 해주세요. 그렇게 해주시면 변경이 용이하겠죠.
  25. 화면 해상도. 와이드 모니터를 생각한다면 배경은 간단한 패턴만 사용했으면 합니다.
    사진 배경으로 할 수 밖에 없다면 1680px 1920px 이상도 고려했으면 합니다.
    ※ InternetTrend - 분석리포트 - 방문자 환경 - 화면해상도 ))) gs.statcounter.com
  26. 중앙 배치 디자인은 좌·우 배경(여백)의 가로 길이를 동일하게 해주세요. 그렇지 않으면 이미지 커팅할 때 디자인 보정 작업까지 해야합니다.
    전체폭 2000px = 좌500px + 웹사이트1000px + 우500px
    좌
    500px    		 웹사이트
    1000px    		 우
    500px
  27. 탭 이미지 등 활성, 비활성 상태를 가지는 이미지는 상황에 따라 2가지 디자인만으로도 가능하게 해주세요.
    3가지 이상의 상태를 가지면 자바스크립트 개발 공정이 추가됩니다.
  28. 벡터는 비트맵으로 변환해주시는 것이 이미지 커팅 작업에 좋습니다.
    벡터 도형 만들 때 경계면에 얼룩이 발생하지 않도록 해주세요.
  29. 텍스트 레이어는 살려주시는게 좋겠습니다.
  30. 포토샵 - 레이어 팔레트 - 눈표시로 확인해야하는 것은 알기쉽게 Layer (Set|Group) Properties - Color 를 지정해주세요.
    (ex) 기본, 선택 상황별, 이벤트별 활성/비활성 콘텐츠, ..
    (ex) 전체메뉴, 바로가기 여닫기, 섹션별 비쥬얼 콘텐츠, ..
    (Tip) 콘텐츠 단위로 이미지 커팅하므로 Layer Set|Group 숨김, 보임을 자주 사용.
    (Tip) 미사용 레이어는 삭제.
  31. line-height 를 font-size+3px 이상되게 디자인해주세요.
    font-size:12px; 의 최소 line-height:15px;
    font-size:11px; 의 최소 line-height:14px;
  32. 포토샵의 텍스트 문단 너비 보다 크게 표현하는 브라우저가 있으므로, 여백을 두고 여유 있게 디자인 바랍니다.
    반응형 웹에서는 더욱 유념하시기 바랍니다.
  33. 레이어 겹침 효과 Normal 이외(Multiply, Overlay 등)를 적용하려면 만들 때 콘텐츠가 겹치지 않도록 해주세요.
  34. Layer Set|Group 4단계 이상 깊이는 지양해주세요. (최대 4단계 허용)
    예) 1단계-전체레이아웃, 2단계-단위콘텐츠, 3단계-각요소. 4단계-각요소의 디자인묶음?

/+visual/

  35. WSG, 가이드를 정해서 디자인 해주시기 바랍니다.
    나쁜예) 글꼴, 버튼, 표, 라인, 그림자, 양각, 음각 스타일이 추가 페이지 디자인 하면서 달라진다.
  36. 로테이션 콘텐츠의 제어 버튼 디자인은 [정지][재생][이전][다음] 순으로 통일해주세요. 같은 페이지에서 순서를 다르게 하지 마세요.
