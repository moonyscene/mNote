운영체제 설치

    
운영체제 설치편과 서버구축 편에서는 가상머신을 만들고, 거기에 운영체제를 직접 설치하고,
그 위에 Apache, MySQL, PHP를 설치하는 방법에 대해서 상세하게 알아본다.

가상머신
가상머신을 이용하면 윈도우가 설치된 컴퓨터에 또 다른 윈도우나 리눅스, Mac OS를 동시에 실행시켜서 마치 여러 대의 컴퓨터를 사용하고 있는 것과 같은 효과를 얻을 수 있다.
virtualbox는 윈도우, MacOS, 리눅스에서 다 사용할 수 있는 크로스플랫폼 프로그램이다.
VMware Player는 virtualbox를 대체 할 수 있는 무료 소프트웨어다. 윈도우와 리눅스에서만 사용 할 수 있다.

VirtualBox & Ubuntu(우분투) 설치

    
1. virtualbox 를 다운로드 & 설치한다. [m] VirtualBox-4.3.8-92456-Win.exe
2. ubuntu desktop 12.x 버전을 다운로드 받는다. [m] ubuntu-12.04.4-desktop-i386.iso

    
3. virtualbox를 실행하고, '새로 만들기'를 실행한다.
4. 가상 머신의 이름과 운영체제의 종류에 (예를들면) '생활코딩 웹서비스 만들기 실습'이라고 입력하고,
    운영체제 항목은 'Linux'를 선택하고, 버전은 'Ubuntu'를 선택한다. [m] Ubuntu(32bit)
5. 메모리는 최소 512MB이상 붉은 색 게이지 이하로 지정한다. [m] 1024MB
6. 가상 하드디스크 항목 중 '부팅 하드디스크'를 체크하고, '새 가상 디스크 만들기'를 선택한다.
    [m] '지금 가상 하드 드라이브 만들기' 선택하고 [만들기]클릭
7. 가상 디스크 만들기에서 파일형식으로 'VDI' 선택
    [m] 하드 드라이브 파일 종류 : VDI (VirtualBox 디스크 이미지) 선택하고 [다음]클릭
8. 가상 디스크 저장소 설정에서 '동적 할당'
    [m] 물리적 하드 드라이브에 저장 : '동적 할당'	선택하고 [다음]클릭 
9. 가상 디스크 파일 위치와 크기. '8GB'
    [m] 파일 위치 및 크기 : '8GB' 선택하고 [만들기]클릭
10. '생활코딩 웹서비스 만들기 실습' 항목 위에서 오른쪽 클릭을 해서 컨텍스트 메뉴를 실행시키고 '설정'을 선택
11. 설정 화면에서 '저장소' 선택 후에 CD 선택
    [m] 저장소 트리에서 '(CD아이콘) 비어 있음' 선택하고,
	속성에서 CD/DVD 드라이브 (IDE 세컨더리…선택상자) 우측 (CD아이콘) 클릭하여,
	'가상 CD/DVD 디스크 파일' 선택 후 [확인]클릭
12. 상기 2번 항목에서 다운로드 받은 파일(ubuntu-12.04-desktop-i386.iso)을 선택
    [m] ubuntu-12.04.4-desktop-i386.iso
13. '생활코딩 웹서비스 만들기 실습' 항목에서 우클릭하여 컨텍스트 메뉴 중에 '시작' 항목을 선택
    [m] 가상 머신 창에 Ubuntu 12.04 … 화면이 보이면서.. 텍스트가 쫙 지나감.. 뭔가 1가지 Fail 후..
	우분투가 작동하면서 바탕화면에 Install 창이 나타난다..

    
14. Install 창 왼쪽의 사이드 바 중에 '한국어'를 선택하고 'Ubuntu 설치' 버튼을 눌러서 설치를 시작한다.
15. 설치 창 Ubuntu 설치 준비 중 화면에서 기본 설정 그대로 '계속' 버튼을 누른다.
16. 설치 창 설치 형식 화면에서 '디스크를 지우고 Ubuntu 설치'를 선택한 후에 '계속'버튼을 누른다.
17. 설치 창 디스크를 지우고 Ubuntu 설치 화면에서 '지금 설치'를 누른다.
18. 설치 창에 installing…바가 보이면서 설치가 시작된다. [m] ▶파일 복사 중...
19. 설치가 진행되는 과정에서 '어디에 살고 계신가요?'를 물어보면 '서울'을 선택하고 '계속'버튼을 누른다.
20. 키보드 배치 화면에서는 '한국어'를 선택하고, '(101/104키 호환)' 항목을 선택하고 '계속'버튼을 누른다.
21. 당신은 누구십니까? 화면에서 필요한 사항을 입력하고 '계속'버튼을 누른다.

22. 설정이 시작된다. [m] ▶설치 중 (몇 십분 진행된다..) … ▶언어 패키지 설치 중  ▶하드웨어 설정 중... ▶ …

    
설치가 완료 된 후에 우분투를 리부팅하게 되는데 이 때 인스톨 화면으로 계속 돌아오는 문제가 있다면 위의 11번 과정에서 삽입한 가상 시디를 꺼내야 한다.

    
    [m] 설치 완료 창에서 [지금 다시 시작]클릭.

24. 우분투 설치가 끝났으면 로그인해서 들어간다.
25. virtualbox에서 제공하는 guest 확장을 설치한다. 게스트 확장 도구는 호스트 운영체제와 게스트 운영체제 간에 클립보드 공유, 해상도 동적 변경, 공유 폴더와 같은 기능을 위해서 사용하는 것이다. 

    
26. virtualbox 윈도우 상단 메뉴에서 [머신]-[게스트 확장 설치]를 실행한다.

29. 게스트 확장의 설치가 실행되면 아래와 같은 화면이 나타난다. 기다린다.
    [m] 터미널 창 (VirtualBox Guest Addintions installration)

31. 화면 상단 우측의 톱니바퀴 모양의 아이콘을 클릭하고, '로그아웃...' 항목을 선택한 후에 다시 로그인을 하면 virtualbox의 창 크기에 따라서 우분투의 화면크기가 능동적으로 변경된다.

리눅스에 Apache, MySQL, PHP 설치
리눅스(우분투)에서 terminal 실행. 터미널 프롬프트는 m@m-VirtualBox:~$ 이런 모양.
터미널에서 아래 명령어 실행하여 패키지를 새 버전으로 업데이트한다.

sudo apt-get update;

sudo는 관리자의 권한으로 따라오는 명령어를 실행하겠다는 의미다.

    
※ 리눅스에서 복사는 [Ctrl+Insert], 붙여넣기는 [Shift+Insert] 단축키를 사용한다.
[m] 계속해서 터미널에서 명령어 입력하여 실행한다.

    
이제 apache2를 설치한다. (리눅스가 알아서 서버에 접속해서 apache2를 다운로드 받고 컴퓨터에 설치할 것이다.)

sudo apt-get install apache2;

mysql 인증 모듈을 설치한다.

sudo apt-get install libapache2-mod-auth-mysql

mysql을 설치한다.

sudo apt-get install mysql-server mysql-client

설치 도중 나타나는 mysql root user password 입력 창에 암호를 입력한다.
실습을 위해서 111111을 입력하도록 하자. (실제 서비스에서는 절대로 이렇게 단순한 비밀번호를 사용해서는 안 된다)
--[m]'암호'() 로 했다가 PHP 실습을 위해서 변경해야 하는데.. 실해해서.. 나중에 다시 해보자
root는 수퍼유저를 의미하는데 모든 작업을 다 할 수 있는 절대권력의 사용자를 의미한다.
PHP를 설치한다.

sudo apt-get install php5-common php5 libapache2-mod-php5

PHP와 MySQL을 연동하기 위한 모듈을 설치한다.

sudo apt-get install php5-mysql

apache를 재시작한다.

sudo /etc/init.d/apache2 restart

mysql 을 재시작한다.

sudo /etc/init.d/mysql restart

잘 설치 됐는지 테스트해보기 위해서 아래의 명령을 입력한다.

sudo gedit /var/www/phpinfo.php

만약 에디터가 실행되지 않는다면 우분투를 재시작해본다. 명령어 입력창에 'sudo reboot'라고 입력하면 된다.
gedit는 GUI 환경에서만 사용할 수 있기 때문에 명령어 환경에서는 nano와 같은 에디터를 사용한다.

sudo nano /var/www/phpinfo.php

아래 문구를 삽입하고 저장한다.

<?php
phpinfo();
?>

파이어폭스를 실행하고 주소 표시줄에 아래와 같이 입력한다.

http://localhost/phpinfo.php

PHP Info 화면이 출력된다면 PHP와 Apache 설치에 성공한 것이다.
이번에는 MySQL을 잘 설치했는지 확인하기 위해서 아래와 같이 입력해보자.

mysql -uroot -p

[m] mysql -u root -p 해도 되더라.
비밀번호를 물어보면 mysql을 설치할 때 입력한 비밀번호를 입력한다.
터미널에서 아래와 같은 화면을 보게 된다면 mysql의 설치에 성공한 것이다.

mysql>

mysql> 상태에서 빠져나올 때는 exit; 를 입력하고 엔터를 치면 된다.
여기까지 성공했다면.. 직접 리눅스를 설치하고, 웹서비스를 하기 위한 준비를 모두 마친 것이다.

버전관리

    
버전관리란?
버전(version)은 변경된 이력을 의미한다.
버전관리 시스템은 코드의 변화를 관리하는 시스템이다.
각각의 변화들을 저장해서 현재 코드의 상태가 어떤 맥락을 거쳐서 지금의 상태가 되었는지를 파악하고,
문제가 생겼을 때 과거로 돌리는 기능 등을 제공한다.
백업의 기능도 하게 된다.
또, 여러 사람이 협업을 할 때 발생할 수 있는 충돌이나 유실을 방지하는 역할도 휼룡하게 수행한다.
버전관리는 프로그래밍에 있어서 빼놓을 수 없는 체계로 자리 잡고 있다.

버전관리는 습관이고, 이 습관은 미래의 당신을 여러번 위기에서 구해줄 것이다. 그러니 꼭 버전관리하자.

    
아무리 생각해도 버전관리는 초심자에게는 어렵다. 당신이 이미 프로그래머이거나, 도전정신이 투철하다면 이번 수업을 따라와도 좋다. 하지만 처음 개발에 도전하는 입장이라면 이번 수업은 시도는 하더라도 막히는 부분이 있다면 과감하게 '클라이언트 측 기술'로 점프하자. 그리고 개발이 어느 정도 손에 익었을 때, 또는 실제 프로젝트를 시작할 때 다시 버전관리를 찾아와도 좋다.

| 테이블 생성

CREATE TABLE topic (
    id  int(11) NOT NULL AUTO_INCREMENT,
    title  varchar(255) NOT NULL ,
    description  text NULL ,
    created  datetime NOT NULL ,
    PRIMARY KEY (id)
);

각 구문의 의미는 아래와 같다.
topic 테이블을 생성한다.
id 컬럼은 숫자형이고 조회시 11자리까지 보여준다. 입력필수(값을 반듯이 입력해야 한다). 값을 특정하지 않으면 자동으로 가장 큰 값이 생성된다. (중복방지)
title 칼럼은 가변형 텍스트이고 최대 255자리까지 입력된다. 입력필수.
description은 길이 제한이 없는 텍스트 형이고 입력은 옵션이다.
created은 2012-04-04 06:16:51와 같은 시간의 형식이 오고, 입력필수다.
id는 유일무이한 값이어야 한다.
';'는 명령이 끝났다는 것을 알려준다. ';'를 사용하지 않으면 명령의 실행이 유보된다.

PHP와 데이터베이스

    
PHP 파일 내에 데이터베이스를 제어하는 구문이 있을 경우 PHP는 아래와 같이 동작한다.

      

    1. 데이터베이스 서버에 접속한다.
      2. 

    2. 데이터베이스를 선택한다.
      3. 

    3. SQL을 데이터베이스로 전송한다.
      4. 

    4. 데이터베이스는 PHP가 전송한 SQL을 실행하고 그 결과를 PHP로 다시 전송한다.
      5. 

    5. PHP는 데이터베이스가 전송한 결과를 이용해 후속 작업을 하거나 HTML 코드 안에 포함시킨다.
      6.

    1. mysql_connect는 데이터베이스에 접속하는 명령이다. 데이터베이스 시간에 MySQL Monitor를 언급했었다. 이 명령은 MySQL Monitor를 실행할 때 인증하는 'mysql -uroot -p' 명령과 같은 역할을 한다. 그렇게 보면 PHP도 MySQL 서버의 입장에서는 클라이언트인 것이다.
