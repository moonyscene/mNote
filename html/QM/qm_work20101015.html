표준 답변
W3C Markup Validator
==> W3C 웹표준 문법은 권고일뿐입니다.
브라우저들이 표준 권고안을 따르긴 하지만 표준을 모두 지원하는 것은 아닙니다.
또한 표준 문법은 권고안마다 달라집니다.
새로운 표준이 이전 문법을 오류로 하거나 오류인 것을 표준으로 받아들이기도 합니다.
브라우저들은 하위 버전과 상위 버전의 호환을 고려하여 렌더링하므로
문법이 조금 다르더라도 호환성에 문제가 발생한다고 볼 수 없으며
문법을 모두 맞췄다고 호환이 해결된다고도 볼 수 없습니다.

그러므로, 변경될지도 모를 문법을 완벽하게 맞추기보다는
가능한 문법에 따라 코딩을 하면서 브라우저 호환을 고려하는 것이 중요합니다.
언어에서 문법이 필요는 하지만 시대에 따라 변하며
지켜야 하는 것은 맞지만 의사소통이 더 중요하다라는 것과 같다고 생각하시면 됩니다.

저희가 개발한 웹문서들은 이런 점들을 잘 고려하여 제작되었고,
W3C Markup(HTML) Validator (http://validator.w3.org/) 결과의 오류들이 호환성 문제를 발생시키지는 않습니다.

행안부 행정기관 웹사이트 진단 결과의 점수는 평가 편의를 위해 기계에만 의존한 것이고,
제대로된 전문가의 컨설팅이 없다면.. 가치 없는 진단 결과일뿐니다.
W3C CSS Validator
==> CSS 2.1 오류 여부로 웹사이트 호환성을 판단하는것 자체가 잘못된 진단입니다.
CSS의 사용에 있어서 중요한 것은 사실상의 표준인 현존하는 웹 브라우저들의 지원 여부에 따르는 겁니다.
그렇게함으로써 최신 웹 브라우저를 선택한 사용자에게는 보다 향상된 경험을 제공할 수 있습니다. 

CSS 문법 유효성 검사 결과는 현실 세계에서 중요하지 않습니다.
만약 누군가 CSS의 문법 유효성을 완벽하게 지켜야 한다고 주장한다면 W3C 웹 사이트의 CSS 소스 코드를 열어보라고 하십시요.
http://jigsaw.w3.org/css-validator/validator?uri=http%3A%2F%2Fwww.w3.org%2F&profile=css21&usermedium=all&warning=1&lang=ko
CSS 문법 오류들이 발견됩니다. 그러나 누구도 W3C가 사용자를 차별했다고 말하지는 않습니다.

브라우저 호환을 위해서라도 오류가 검출되지 않게 할 수 없습니다.

작년의 행안부 웹사이트 진단에서는 아래와 같이..
개별 *.css 파일을 직접 검사하지 않고 css 를 참조하는 웹문서로 CSS 유효성 검사를 하였습니다.

http://jigsaw.w3.org/css-validator/validator?uri=http%3A%2F%2Fwww.busan.go.kr%2F00_main%2F&profile=css21&usermedium=all&warning=1&lang=ko

CSS 유효성 검사를 통과하고 싶으시면.. 오류로 검출되는 인라인 스타일시트를 이 웹페이지에서 참조중인 외부CSS파일(layout.css)에 정의하면 됩니다.
(www.busan.go.kr 사이트 설계할 때 그렇게 만들었습니다.)

PS) 오류로 발견된 ime-mode: active; 는 인풋박스 선택시 자동으로 한글 입력모드로 만들어줍니다.
이것은 ime-mode: 속성을 지원하는 브라우저의 사용성을 향상시키는 것이지,
입력을 못하게 한다든지 하는 호환성 문제는 전혀 없습니다.
