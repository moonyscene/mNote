    ==> codebase 속성을 삭제해도 동영상 나옴.
    [m]20090821.
    Opera9.64 에서 동영상 플레이어는 보이나 동영상을 찾지 못하는 오류가 발생하여 codebase 를 삭제하니 동영상이 재생됨.
    codebase 는 필수 속성이 아니므로 앞으로 사용하지 말자.
    id="player" 는 자바스크립트에서 콘트롤 하기 위해 만든 속성이니 필요하지 않아서 삭제함.
    type 속성과 classid 속성 위치 바꿔서 IE외 표준 코드와 속성 순서 맞춤.
    [m]20120823.
    [2012.08.WA마크18회.진주시] 사용자 심사 - (전맹_죠스) 죠스로 운용 시에는 해당 동영상 제어버튼이 인지, 운용되지 않아 동영상을 재생하지 못함.
    (해결방안) object 태그에 classid와 param값을 적절히 제공함. (class id를 미디어플레이어 6.4 버전 이하로 지정함)
    하지만, mp3, wma 재생 안된다. 다시 7 이상 버전 (classid="clsid:6bf52a52-394a-11d3-b153-00c04f79faa6") 으로 변경

    N [2012.10.12.진주시] [CR] 플러그인이 올바르게 설치되어 있는데도 동영상 플레이어만 보이고 재생이 안됨.
    (해결방안) 동영상 소스 파일을 프로토콜(http:// 또는 mms://)로 시작하는 절대경로로 변경하니 해결됨.
    N [2013.11.12~21.의령군] -- 호환성 평가기관 답변 --
    동영상 관련 문의는 감사원 홈페이지의 홍보동영상 페이지를 확인하시면 될거 같습니다.
    또한 플러그인이 안깔려서 그런것 아니냐는 문의는, 플러그인이 안깔려 있을때 플러그인은 깔수 있는 URL을 제공 혹은 어떠한 플러그인을 깔아야 하는지 안내를 해줘야 하는 사항 입니다.

    20120417. mp3 도 가능하다. type="video/x-ms-wmv" 이 안맞아도 재생된다. [CR] 에서 안될 때 있던데.. 로컬에서는 되더라. type="video/x-ms-wmv" 을 맞추어도 안된다. 서버 설정을 잘 봐야할 듯~.
