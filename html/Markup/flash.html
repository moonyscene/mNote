플래시 콘텐츠 제작시 유의점

플래셔, 콘텐츠디자인코더에게 바란다.

  1. 초당 3~49번 깜박임은 넣지 말아주세요..
    - 나쁜예 : 비쥬얼 플래시가 과도하게 깜박임
  2. 플래시 무비 ActionScript 에 Stage.scaleMode = "noScale"; 가 있으면 화면 확대가 안됩니다.
    - 나쁜예 : 화면 확대 버튼 클릭해보면 비쥬얼 확대 안됨.
  3. 플래시 콘텐츠 자체 접근성 보강 최선의 방법은? 탭인덱스와 대체텍스트를 반드시 지정하자!
    ※ 참고 : 플래시 콘텐츠 자체 접근성 보강
    - 나쁜예 : 배치도 플래시 탭 이동 순서가 뒤죽박죽
  4. 플래시 자체적인 접근성을 준수하거나, 플래시 없는 버전 (HTML, Image)을 제공하여 사용자가 대체 콘텐츠를 선택하여 이용할 수 있어야 한다.
    ※ 플래시 콘텐츠 접근성 제작 방법 등
    ※ HTML버전 제공예 : 의령군 읍면별 주산작물
    플래시 자체 탭키 이동, 대체텍스트 작업 완료하였습니다. 플래시 콘텐츠는 화면낭독프로그램 종류에 따라 접근 여부를 장담할 수 없습니다. 그래서, 화면에는 보이지 않지만 별도의 HTML 버전도 제공하여 화면낭독프로그램 이용자가 선택하여 이용할 수 있게 하였습니다.
  5. 2008.10.29 이후 프로젝트에서 플래시 코딩은 아래의 플래시 기본 샘플 코드를 사용하자!
    이전 방식 코드에서 google 크롬 0.2.149.30 버전에서 플래시 안보이는 것은 클라이언트 요청시에만 수정해주세요.
  6. wmode 가 opaque, transparent 일 경우 IE7,8외에는 키보드 운용 안됨. window 로 하세요.

<param name="wmode" value="window" />
  • window : 기본값으로 모든 html 엘리먼트 위로 flash 가 나타납니다.
  • opaque : flash 위에 html 엘리먼트를 위치 시킬때 사용합니다.
  • transparent : flash 를 투명하게 하여 flash 아래의 html 엘리먼트가 화면상에 나오게 됩니다.

브라우저별 예외 환경 검사 결과 (Screen Reader 시험 추가)

※ 예외 환경이란? 일반 환경(CSS 지원, JavaScript 지원, Flash Plugin 지원)에서 기능이 1가지 이상 지원되지 않는 경우

Disable Plugin 일 때는 대체콘텐츠가 올바로 보이는지(들리는지) 유무 체크

센스리더PE2.2 의 경우 Disable Plugin 일 때만 대체텍스트(object 요소 안쪽의 대체콘텐츠)를 읽어준다.
그러므로, 반드시 플래시 자체적인 접근성을 준수하거나, 그러지 못할 경우
플래시 없는 버전 (HTML, Image)을 제공하여 사용자가 대체 콘텐츠를 선택하여 이용할 수 있어야 한다.

스크린리더 확인

[센스리더 PE2.2]
  • 플래시 탭이동 불가, 플래시 자체 대체콘텐츠 읽어주지 않고 온마우스시에도 안읽음.
  • object 태그의 대체콘텐츠는 플래시 플러그인 제거시에만 읽음.
  • 플래시 플러그인 활성화시 <!--[if !IE]>-->대체텍스트<!--<![endif]--> 읽지 않음.
  • noscript 안의 내용은 읽지 않음.
[드림보이스6.21]
  • 플래시 탭이동 불가, 플래시 자체 대체콘텐츠 읽어주지 않지만 온마우스시에는 읽음.
  • object 태그의 대체콘텐츠는 읽음. (js 파일에서 document.write 해도 읽음.)
  • <!--[if !IE]>-->대체텍스트<!--<![endif]--> 부분도 읽음(가끔 못 읽을 때도 있음).
  • noscript 안의 내용은 읽지 않음.

브라우저별 예외 환경 조성 방법

  • IE6 : IE Developer Toolbar 설치 후 검사.
  • IE7 : IE6 과 동일한 방법 사용. IE Developer Toolbar 의 Disable Script 기능을 사용할 수 없었음.(IE7 for Vista)
  • FF : Web Developer Toolbar 설치 후 검사.
    Plugin Disable 기능은 없으므로 object 요소의 type="" data="" 속성을 제거 후 검사
    (플래시를 안보이게 하는 플러그인 FlashBlock 등이 있으나 대체텍스트를 보여주지는 않는다.)
  • Op : 브라우저에 Disable 기능이 모두 기본으로 제공.
  • Sf : 작성자 CSS Disable 기능이 없고 사용자 CSS 추가만 가능함. 그외 기능은 모두 기본 제공.
    [개발자용] 메뉴에서 제공.. 단, 플러그인 비활성화는 [편집]-[기본설정]에서 제공.

관련정보

  • 플래시에 포함되는 내용에 따라 다릅니다.
  • “Adobe Flash CS4 Professional accessibility”라는 튜토리얼
    주요 구성은 개요(Introduction), 시각장애인을 위한 플래시 디자인, 키보드 접근성, 청각장애인을 위한 플래시 디자인, 참고자료로 구성되어 있습니다.
  • 브라우저에 따라 미세한 차이가 존재한다. 예를 들면 FireFox, Opera, Sapari에서는 object 태그와 param 태그를 이용하는 경우에 Flash 콘텐츠를 표시하지 못하는 경우가 발생한다. 또한
    • Internet Explorer에서는 embed 태그와 param 태그를 함께 설정하지 않으면 JavaScript 함수 호출은 가능하나 함수의 리턴 값을 사용할 수 없는 경우가 발생한다.

    위 소스에서 첫번째 object 요소 다음에 param 요소를 중복으로 적는 방법 또는 웹표준 object 코딩을 사용할 수 있음.
  • wmode 가 "transparent" 일 때 키보드로 플래시 콘텐츠 자체에 접근하는 방법은 없는가?
    현재로선 없으니.. 2008.11.13 현재 IE7이상에서만 가능하므로(FF, Op, Sa 안됨), wmode 는 "window" 를 기본으로 하고,
    플래시 위에 다른 콘텐츠가 올 때는 "opaque" 를 쓰자!
    "transparent" 는 진짜 진짜 플래시 투명 영역 밑으로 다른 콘텐츠가 보여야 할 경우에만 쓰자!
    - 예 : 플래시 배치도 wmode 를 "window" 로도 가능한데 "transparent"를 사용한 경우
